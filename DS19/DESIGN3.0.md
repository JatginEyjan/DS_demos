# DS19 Design 3.0
## Dungeon Spirit 19 - 地牢精灵

**一句话定位**：用波次驱动的宝可梦Roguelike，每局都是实验，每次捕获都是长期收藏。

---

## 一、核心驱动：波次（Wave）

### 1.1 波次即一切

**单局结构**：
```
波次 1 → 2 → 3 → ... → 30（Demo终点）
```

**波次驱动规则**：
| 系统 | 波次规则 |
|------|----------|
| 敌人等级 | `level = 3 + wave * 1.5 + (wave/10)^2` |
| 敌人强度 | 每5波强度跃升（新技能/新机制） |
| Boss出现 | `wave % 10 === 0`（第10/20/30波固定Boss） |
| 商店刷新 | `wave % 5 === 0`（第5/10/15...波后可选进入） |
| 切换点 | 每10波强制选择：继续/撤退/商店 |
| 等级上限 | `cap = 5 + wave * 1.5`（软上限，超等级经验递减） |

### 1.2 为什么用波次而非"层"

**层的假象**：玩家以为"我到了第3层=进度30%"，但实际体验可能是一层很长或很短。

**波次的真相**：每波都是一次独立的挑战-奖励循环。玩家永远知道"还有2波到Boss"，节奏感明确。

## 二、精灵系统：单局实验，长期收藏

### 2.1 单局规则

**开局**：
- 选择1只已解锁的starter精灵
- 携带3个普通精灵球
- 初始金币：100

**精灵上限**：单局最多携带 **6只**（致敬经典，也是决策压力点）

**濒死机制**：
- HP归零=濒死，不可出战
- 濒死精灵的牌从牌库移除
- 单局内**无法复活**（硬核）
- 全部6只濒死=游戏结束

**切换机制（关键设计）**：
| 切换类型 | 代价 | 时机 |
|----------|------|------|
| 主动切换 | 消耗 **1点能量** | 玩家回合任意时刻 |
| 濒死切换 | **无代价** | 强制触发 |
| 战后切换 | 无代价 | 战斗结束后 |

**为什么1点能量而非整回合？**
- 整回合=自杀，玩家永远不会主动切换
- 1点能量=有代价但可承受，让玩家愿意为了"属性克制"或"状态管理"而切
- 濒死无代价=保险机制，防止被系统惩罚

### 2.2 牌库机制

**核心规则**：
- 当前出战精灵的整套技能牌=当前可用牌库
- 切换精灵时：**手牌保留，弃牌堆保留，抽牌堆重置为新精灵的牌库**

**为什么保留弃牌堆？**
- 防止"切换躲避疲劳"的滥用
- 让玩家承担"切换是有代价的"——你可能带着一堆废牌切入新精灵

**疲劳机制**：
- 牌库抽空后，每抽1张受到疲劳伤害
- 疲劳伤害 = 已抽疲劳牌数量（1,2,3...递增）

### 2.3 经验与成长（反优化设计）

**经验获取**：
```
击败敌人获得经验 = enemy.level / 3 + 1
```

**关键设计：移除"等级差加成"**
- 不鼓励玩家"压级打高级敌人"来刷经验
- 每波敌人和玩家等级动态匹配，不存在"越级刷经验"的套利空间

**升级需求（指数曲线）**：
```
当前等级1→2需要：2 EXP
当前等级2→3需要：3 EXP
当前等级3→4需要：4 EXP
...
当前等级n→n+1需要：(n+1) EXP
```

**升级奖励（3选1）**：
- 选项从该精灵的"技能池"中随机抽取
- 稀有度权重：普通60%，稀有30%，史诗10%
- **无保底**——可能3张都是普通，这就是随机性

**等级上限（软墙）**：
- 公式：`cap = 5 + wave * 1.5`
- 超过上限后，经验获取效率降至10%
- 不是硬禁止，而是软引导："该推图了，别刷了"

## 三、捕获系统：心理博弈

### 3.1 捕获率公式

```
基础捕获率 = 30%

HP修正：
- HP > 50%：+0%
- HP 25-50%：+15%
- HP < 25%：+30%

异常状态修正：
- 中毒/麻痹/灼烧：+20%
- 睡眠/冰冻：+40%

球种倍率：
- 普通球：1.0x
- 高级球：1.5x
- 超级球：2.0x
- 大师球：100%（代码特殊处理）

最终捕获率 = min(95%, 基础率 + HP修正 + 状态修正) * 球种倍率
```

### 3.2 捕获流程

1. 战斗中选择"捕获"（不消耗能量，但占用一个"行动"）
2. 选择精灵球
3. 播放3次晃动动画
4. 每次晃动判定：随机数 < 捕获率则通过
5. 3次全过=捕获成功

**关键设计：捕获成功≠战斗结束**
- 捕获成功后，敌人被替换为新的野生精灵（同波次强度）
- 或者选择"撤退"结束战斗
- 为什么？防止"为了捕获而捕获"的功利玩法，保持战斗的完整性

### 3.3 精灵球获取

| 来源 | 数量 |
|------|------|
| 开局 | 3个普通球 |
| 每场战斗胜利 | 10%概率掉落普通球 |
| 商店 | 购买 |
| Boss击败 | 固定掉落2个高级球 |

**经济设计意图**：球是稀缺资源，玩家必须决策"这只精灵值不值得抓"。

## 四、战斗数值：模块化乘区

### 4.1 伤害公式

```
damage = baseDamage * typeMultiplier * critMultiplier * randomMultiplier * burnMultiplier

baseDamage = (movePower * user.ATK / target.DEF / 5) + 2
```

### 4.2 各乘区详解

**属性克制（typeMultiplier）**：
```
克制：2.0x
被克：0.5x
无效：0x（免疫）
```

**暴击（critMultiplier）**：
| 暴击阶段 | 概率 | 倍率 |
|----------|------|------|
| 0阶 | 1/24 | 1.5x |
| 1阶 | 1/8 | 1.5x |
| 2阶 | 1/2 | 1.5x |
| 3阶+ | 100% | 1.5x |

暴击阶段来源：技能特效、装备、能力

**随机浮动（randomMultiplier）**：
- 范围：0.85 ~ 1.00
- 目的：让战斗有叙事性，记住"那次差点打死"的时刻

**状态影响**：
- 灼烧：物理伤害x0.5
- 麻痹：20%概率无法行动

### 4.3 能力阶段（指数曲线）

| 阶段 | ATK/DEF/SPD倍率 |
|------|-----------------|
| -3 | 0.4x |
| -2 | 0.5x |
| -1 | 0.67x |
| 0 | 1.0x |
| +1 | 1.5x |
| +2 | 2.0x |
| +3 | 2.5x |
| +6 | 4.0x（上限） |

**设计意图**：
- +1有明显收益（鼓励使用buff）
- +6有巨大回报（值得投资）
- -3几乎废掉（debuff风险极高）

## 五、进度系统：多线交织

### 5.1 单局进度（重置）
- 波次推进
- 精灵等级
- 装备获取
- 金币积累

### 5.2 收藏进度（永久）

**图鉴系统**：
- 捕获新精灵=解锁图鉴
- 图鉴数量影响"临界捕获"概率（DS20同款设计）

**糖果系统**：
- 每场战斗：出战的精灵+3好感度
- 濒死：-5好感度
- 好感度达到阈值=获得该精灵的"糖果"
- 糖果用途：解锁该精灵作为starter（开局可选）

**好感度阈值表**：
| 精灵稀有度 | 解锁所需好感度 |
|------------|----------------|
| 普通 | 50 |
| 稀有 | 100 |
| 史诗 | 200 |
| 传说 | 400 |

**设计意图**：
- 鼓励玩家使用不同精灵
- 即使捕获了传说精灵，也需要"带它战斗"才能解锁开局使用
- 形成"玩→收集→解锁→再玩"的闭环

### 5.3 成就/挑战（长期）
- 无伤击败Boss
- 单精灵通关
- 全属性队伍通关
- 捕获闪光精灵（1/256概率）

## 六、商店与经济

### 6.1 金币获取

| 来源 | 数量 |
|------|------|
| 击败普通敌人 | 10 + wave |
| 击败精英敌人 | 30 + wave * 2 |
| 击败Boss | 100 + wave * 5 |

### 6.2 商店机制

**触发**：`wave % 5 === 0` 后，玩家可选择进入商店或继续

**商品池**：
| 商品 | 价格 | 出现概率 |
|------|------|----------|
| 普通球 | 50 | 30% |
| 高级球 | 150 | 20% |
| 伤药（30HP） | 40 | 25% |
| 装备（随机） | 100-300 | 15% |
| 移除卡牌服务 | 75 | 10% |

**设计意图**：
- 每5波强制让玩家决策："我现在的状态值得投资，还是该存钱了？"
- 移除卡牌服务让玩家可以精简牌库（DBG核心乐趣）

## 七、Boss设计（机制驱动）

### 7.1 Boss出现规则
- `wave % 10 === 0`
- 第10波：第一层Boss
- 第20波：第二层Boss
- 第30波：最终Boss

### 7.2 Boss机制设计原则
**不是"血多攻高"，而是"改变游戏规则"**

#### 第10波 Boss：属性守望者
**机制**：
- 每3回合切换属性（火→水→草→循环）
- 当前属性下，同属性伤害-50%，克制属性+50%
- 切换时清除自身所有debuff

**应对策略**：携带多属性精灵，预判切换时机

#### 第20波 Boss：幻影分身
**机制**：
- 开场召唤2个分身（分身血量=本体30%）
- 分身会复制本体的攻击
- 只有击败本体才算胜利
- 每4回合重新召唤消失的分身

**应对策略**：AOE技能或快速击杀本体

#### 第30波 Boss：混沌吞噬者
**机制**：
- 每回合随机"禁用"一个属性（该属性精灵无法造成伤害）
- 禁用持续2回合
- 每5回合释放全屏高伤害技能（必须提前预留防御牌）

**应对策略**：队伍属性多样化，预留防御/回复手段

### 8.2 技能设计框架

每只精灵：
- 初始牌库：4张打击 + 4张防御
- 可学习技能池：8-12张
- 升级时3选1

**技能稀有度分布**：
- 普通技能：60%出现概率（基础伤害/防御）
- 稀有技能：30%出现概率（带特效：灼烧/麻痹/抽牌等）
- 史诗技能：10%出现概率（强力效果但有代价）

## 九、设计哲学总结

### 9.1 核心循环
```
选择starter → 推波次 → 遇敌决策（打/抓/跑）→ 升级选技能 → 
构建牌库 → 挑战Boss → 捕获新精灵 → 积累好感 → 
解锁新starter → 下一局尝试新build
```

### 9.2 关键设计决策

| 决策 | DS19 v3.0 | 为什么 |
|------|-----------|--------|
| 切换代价 | 1点能量 | 有代价但可承受，鼓励策略切换 |
| 濒死复活 | 单局不可复活 | 增加决策重量，让每场战斗有意义 |
| 经验公式 | 无等级差加成 | 反优化，防止功利刷级 |
| 捕获成功 | 不结束战斗 | 防止为抓而抓，保持战斗完整性 |
| 软等级上限 | 超上限经验递减 | 引导推图，而非硬限制 |
| 波次驱动 | 一切由wave决定 | 节奏明确，玩家始终知道进度 |

### 9.3 从DS20学到的精髓

1. **单一变量驱动**：波次决定一切，节奏清晰
2. **可控的失控**：随机性分层，确定性与惊喜并存
3. **反玩家优化**：主动关闭"刷经验"等痛苦优化
4. **单局实验场**：每局重置，但长期进度有意义
5. **软上限设计**：引导而非强制，保持玩家掌控感
6. **指数曲线**：能力阶段的非线性增长创造决策张力
7. **心理博弈**：捕获系统的紧张感设计
