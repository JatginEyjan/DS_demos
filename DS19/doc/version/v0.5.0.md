# v0.5.0 - 角色招募与替换

## 版本目标
验证阵容经营机制：中途获得新角色、等级继承、替换决策。

---

## 功能范围

### 必须实现 {#must-have}

- [ ] **4名完整角色**
  - 引用：[角色清单](../03_角色与卡牌清单.md#character-list)
  - 验收：
    - 战士：防御/承伤定位
    - 游侠：连击/易伤定位
    - 法师：爆发/过牌定位
    - 祭司：恢复/支援定位
    - 各有完整基础牌库+成长牌池

- [ ] **角色招募**
  - 引用：[新角色获取](../01_Demo玩法规则.md#new-character), [招募价格](../02_Demo数值表.md#recruit-price)
  - 验收：
    - 商店可招募新角色（90金币）
    - 事件可获得新角色
    - 队伍未满时直接加入

- [ ] **角色替换**
  - 引用：[替换规则](../01_Demo玩法规则.md#replace-rule), [替换继承](../02_Demo数值表.md#replace-inherit)
  - 验收：
    - 队伍满员时触发替换选择
    - 显示两名当前角色供选择替换
    - 可选择放弃新角色

- [ ] **等级继承**
  - 引用：[等级继承](../02_Demo数值表.md#level-inherit)
  - 验收：
    - 新角色继承被替换角色的等级
    - Lv2继承：基础牌库+1张成长牌
    - Lv3继承：基础牌库+2张成长牌
    - 成长牌由系统随机从卡池选择

- [ ] **初始角色选择**
  - 引用：[开局规则](../00_GDD.md#start-rule)
  - 验收：
    - 游戏开始可选择战士或游侠作为首发
    - 清晰展示角色定位

---

## 接口契约 {#interface-contract}

### 招募系统输入依赖
- 当前队伍状态
- 新角色配置
- 金币（商店招募时）

### 招募系统输出
- 更新后的队伍
- 扣除的金币

### 替换系统输出
- 新角色（继承等级）
- 移除的旧角色

---

## 验收标准 {#acceptance}

### 单元验收
- [ ] 4名角色各有独特基础牌库
- [ ] 商店招募消耗90金币
- [ ] 替换正确继承等级
- [ ] 替换后新角色牌库符合等级

### 集成验收（关键）
- [ ] **阵容转型**：Lv3战士替换为Lv3法师，法师有3张成长牌
- [ ] **双角色协同**：战士+游侠/战士+法师/游侠+祭司等组合各有玩法
- [ ] **替换决策有意义**：替换是重大决策，有得有失

### 体验验收
- [ ] 4名角色差异明显
- [ ] 不同组合有不同体验
- [ ] 替换不会让人感到"亏了"
