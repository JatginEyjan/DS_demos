# v0.2.0 - 升级与成长系统

## 版本目标
验证"升级选卡"机制：战斗获得经验 → 升级触发三选一 → 新卡加入牌库 → 下一场战斗使用新牌库。

---

## 功能范围

### 必须实现 {#must-have}

- [ ] **经验系统**
  - 引用：[经验获取](../02_Demo数值表.md#exp-gain), [战后经验规则](../01_Demo玩法规则.md#post-battle-exp)
  - 验收：
    - 普通战斗胜利：+1 XP
    - 精英战斗胜利：+2 XP
    - Boss战斗胜利：+3 XP
    - 死亡角色仍获得经验

- [ ] **升级判定**
  - 引用：[升级需求](../02_Demo数值表.md#level-up-requirement)
  - 验收：
    - Lv1→2: 2 XP
    - Lv2→3: 3 XP（累计5）
    - Lv3→4: 4 XP（累计9）
    - Lv4→5: 5 XP（累计14）

- [ ] **三选一界面**
  - 引用：[升级抽卡规则](../01_Demo玩法规则.md#level-up-card)
  - 验收：
    - 升级时弹出选卡界面
    - 显示3张候选卡牌
    - 卡牌显示费用、效果、稀有度
    - 点击选中后关闭界面

- [ ] **卡牌加入牌库**
  - 引用：[牌库归属](../01_Demo玩法规则.md#deck-ownership)
  - 验收：
    - 选中卡牌加入对应角色牌库
    - 下一场战斗新卡牌出现在抽牌堆
    - 升级时最大生命+4（战士+5）
    - 升级时当前生命+8

- [ ] **等级上限**
  - 引用：[等级上限](../02_Demo数值表.md#level-cap)
  - 验收：Lv5为上限，不再获得经验

### 角色范围

| 角色 | 可用成长牌 | 说明 |
|------|------------|------|
| 战士 | 6张普通 + 3张进阶 + 1张稀有 | 从完整卡池抽取 |
| 游侠 | 6张普通 + 3张进阶 + 1张稀有 | 从完整卡池抽取 |

## 接口契约 {#interface-contract}

### 升级系统输入依赖
- 战斗结果（来自战斗系统）
- 角色当前经验/等级（来自角色状态）
- 角色成长卡池（来自角色配置）

### 升级系统输出
- 更新后的角色等级
- 更新后的角色牌库
- 更新后的角色最大生命

### 关键数据流
```
战斗胜利 ──→ 经验结算 ──→ 升级判定 ──→ 三选一界面
                              ↓
                         新卡加入牌库
```

## 验收标准 {#acceptance}

### 单元验收
- [ ] 战士获得2 XP后升级至Lv2
- [ ] 升级界面正确显示3张候选卡
- [ ] 选中卡牌正确加入战士牌库
- [ ] 升级后战士最大生命+5，当前生命+8
- [ ] 游侠升级逻辑同理

### 集成验收（关键）
- [ ] **成长验证**：升级获得的卡牌在下一场战斗中可被抽到
- [ ] **多升级验证**：一次战斗获得足够经验连升2级，触发两次三选一
- [ ] **双角色独立**：战士升级不影响游侠，游侠升级不影响战士

### 体验验收
- [ ] 升级反馈明确（界面+音效）
- [ ] 三选一决策有意义（3张卡各有不同价值）
- [ ] 成长感可感知（牌库逐渐变大）
