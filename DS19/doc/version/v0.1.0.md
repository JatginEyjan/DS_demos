# v0.1.0 - 核心战斗验证

## 版本目标
验证双角色战斗循环的基础可行性：两名角色同时在场、独立资源、自由切换、回合循环正确运行。

---

## 功能范围

### 必须实现 {#must-have}

- [ ] **双角色在场**
  - 引用：[队伍系统](../00_GDD.md#party-system)
  - 验收：两名角色同时显示在战斗场景中

- [ ] **独立资源系统**
  - 引用：[资源系统](../00_GDD.md#resource-system), [角色属性](../02_Demo数值表.md#character-stats)
  - 验收：
    - 角色A的能量消耗不影响角色B
    - 角色A的抽牌来自A的牌库，角色B同理
    - 手牌区域明确区分归属

- [ ] **回合循环**
  - 引用：[回合流程](../01_Demo玩法规则.md#turn-flow)
  - 验收：
    - 玩家回合开始：回复能量+抽牌
    - 玩家行动：出牌/切换/结束
    - 玩家回合结束：弃牌+格挡清除
    - 敌方回合：执行意图
    - 循环回玩家回合

- [ ] **角色切换**
  - 引用：[切换规则](../01_Demo玩法规则.md#switch-rule)
  - 验收：
    - 点击切换按钮/快捷键可切换当前操作角色
    - 切换不消耗能量
    - 切换后手牌区域更新为当前角色手牌
    - 当前角色有明显高亮标识

- [ ] **基础卡牌系统**
  - 引用：[基础牌](../03_角色与卡牌清单.md#base-deck), [STS卡牌基线](../STS_REFERENCE.md#card-baseline)
  - 验收：
    - 1费攻击牌：造成6伤害
    - 1费防御牌：获得6护甲
    - 卡牌可拖拽/点击打出
    - 能量不足时卡牌不可用（灰显）

- [ ] **敌人意图系统**
  - 引用：[意图显示](../02_Demo数值表.md#intent-system)
  - 验收：
    - 敌人上方显示下回合行动意图
    - 显示伤害数值
    - 显示攻击目标（角色A或B）

- [ ] **胜负判定**
  - 引用：[胜负规则](../01_Demo玩法规则.md#win-lose)
  - 验收：
    - 敌人全灭 → 胜利
    - 双角色死亡 → 失败
    - 单角色死亡 → 战斗继续

### 角色与敌人范围

| 类型 | 数量 | 说明 |
|------|------|------|
| 可用角色 | 2 | 战士 + 游侠（仅基础牌库） |
| 敌人类型 | 1 | 标准攻击型敌人（HP: 35, 伤害: 8-11） |
| 战斗场景 | 1 | 单战斗场景，无地图 |

## 接口契约 {#interface-contract}

### 战斗系统输入依赖
- 角色数据（来自角色配置）
- 卡牌定义（来自卡牌数据表）
- 敌人配置（来自敌人数据表）

### 战斗系统输出
- 战斗结果（胜利/失败）
- 角色最终状态（HP等）

### 关键数据流
```
角色配置 ──→ 战斗系统 ←── 敌人配置
     │              ↓
     └──→ 卡牌实例 ←──┘ 战斗逻辑
```

## 验收标准 {#acceptance}

### 单元验收
- [ ] 战士可以打出4张Strike+4张Defend
- [ ] 游侠可以打出4张射击+4张闪避
- [ ] 双角色能量独立计算（各3点/回合）
- [ ] 双角色手牌独立（各4张/回合）
- [ ] 切换角色后当前操作角色正确更新
- [ ] 敌人意图正确显示
- [ ] 伤害计算符合公式：(基础+力量)×易伤×虚弱-格挡

### 集成验收（关键）
- [ ] **双角色协同场景**：战士防御+游侠输出完成一场战斗
- [ ] **切换流畅度**：10次切换无卡顿、无状态错乱
- [ ] **残局场景**：战士死亡后，游侠单独完成剩余战斗

### 体验验收
- [ ] 切换操作直观，3秒内理解
- [ ] 手牌归属清晰，不会混淆
- [ ] 战斗节奏符合预期（4-6回合结束）

## 阻塞风险

| 风险 | 概率 | 应对 |
|------|------|------|
| 双角色UI信息过载 | 中 | 若验收失败，减抽牌数至3张/角色 |
| 切换手感不佳 | 低 | 增加视觉反馈（高亮动画）|
| 牌库洗牌Bug | 中 | 重点测试抽光牌库后的行为 |
