# v0.3.0 - 地图与流程

## 版本目标
验证完整单局流程：从起点出发 → 经过多个节点 → 抵达Boss → 击败完成章节。

---

## 功能范围

### 必须实现 {#must-have}

- [ ] **地图场景**
  - 引用：[地图推进](../01_Demo玩法规则.md#map-progress)
  - 验收：
    - 显示节点网络（8-10个节点）
    - 节点有不同图标（战斗/精英/商店/事件/Boss）
    - 已访问节点标记
    - 可点击下一个可选节点

- [ ] **节点类型**
  - 引用：[节点类型](../00_GDD.md#node-types), [房间类型](../02_Demo数值表.md#room-types)
  - 验收：
    - ⚔️ 普通战斗：1-3个敌人
    - 👹 精英战斗：1个强敌
    - 👑 Boss：1个章节Boss
    - 🔥 篝火：恢复/升级二选一
    - ❓ 事件：简单文本事件

- [ ] **路线选择**
  - 引用：[路线规划](../00_GDD.md#map-route)
  - 验收：
    - 从起点到Boss有多条路径
    - 玩家可选择不同路线
    - 已走过的节点不可重复访问

- [ ] **战后结算**
  - 引用：[战后结算](../01_Demo玩法规则.md#post-battle)
  - 验收：
    - 战斗胜利后显示奖励（金币+经验）
    - 触发升级时进入三选一
    - 完成后返回地图

- [ ] **章节Boss**
  - 引用：[Boss数值](../02_Demo数值表.md#boss-stats)
  - 验收：
    - Boss HP: 180-220
    - 常规伤害: 12-18
    - 爆发伤害: 22-30（有预兆）
    - 击败后显示胜利画面

### 地图配置

| 节点类型 | 数量 | 说明 |
|----------|------|------|
| 起点 | 1 | 固定位置 |
| 普通战斗 | 4-5 | 主要经验来源 |
| 精英战斗 | 1-2 | 高风险高奖励 |
| 篝火 | 1-2 | 恢复生命或升级 |
| 事件 | 1-2 | 简单随机事件 |
| Boss | 1 | 章节终点 |

## 接口契约 {#interface-contract}

### 地图系统输入依赖
- 地图配置（节点布局）
- 战斗结果（来自战斗系统）
- 角色状态（HP、等级等）

### 地图系统输出
- 进入战斗（携带当前角色状态）
- 进入事件
- 进入商店
- 章节完成

### 关键数据流
```
地图场景 ──→ 选择节点 ──→ 进入战斗/事件/商店
                ↓
         战后返回地图 ──→ 继续选择
```

## 验收标准 {#acceptance}

### 单元验收
- [ ] 地图正确显示所有节点
- [ ] 点击节点可进入对应场景
- [ ] 战后正确返回地图
- [ ] 角色状态在节点间保持（HP、等级、牌库）

### 集成验收（关键）
- [ ] **完整流程**：从起点走到Boss并击败
- [ ] **路线差异**：不同路线遇到的战斗数量不同
- [ ] **状态保持**：战斗受伤后，下一场战斗继承当前HP
- [ ] **升级生效**：地图层触发的升级，下一场战斗生效

### 体验验收
- [ ] 路线选择有意义（风险vs奖励）
- [ ] 流程长度适中（15-20分钟）
- [ ] Boss战有挑战性

---

## 不包含在此版本

- ❌ 商店系统（篝火只有恢复/升级）
- ❌ 复杂事件（仅简单文本）
- ❌ 角色招募/替换
- ❌ 多章节
