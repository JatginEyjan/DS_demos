# v0.4.0 - 商店与事件系统

## 版本目标
验证资源管理决策：金币获取与消费、商店购买选择、事件风险收益权衡。

---

## 功能范围

### 必须实现 {#must-have}

- [ ] **金币系统**
  - 引用：[金币奖励](../02_Demo数值表.md#gold-reward)
  - 验收：
    - 普通战斗：+18金币
    - 精英战斗：+35金币
    - Boss战斗：+60金币
    - 金币在章节间保持

- [ ] **商店界面**
  - 引用：[商店规则](../01_Demo玩法规则.md#shop-rule), [商店价格](../02_Demo数值表.md#shop-price)
  - 验收：
    - 显示可购买商品列表
    - 显示当前金币
    - 点击购买扣除金币
    - 离开商店返回地图

- [ ] **商店商品**
  - 引用：[商店价格](../02_Demo数值表.md#shop-price)
  - 验收：
    - 恢复20生命：18金币
    - 删除1张卡牌：45金币
    - 普通卡牌：35金币
    - 进阶卡牌：55金币
    - 稀有卡牌：80金币
    - 普通遗物：75金币
    - 高价值遗物：110金币

- [ ] **删除卡牌**
  - 引用：[商店规则](../01_Demo玩法规则.md#shop-rule)
  - 验收：
    - 选择要删除的卡牌
    - 删除后从牌库移除
    - 基础牌（Strike/Defend）也可删除

- [ ] **事件节点**
  - 引用：[事件规则](../01_Demo玩法规则.md#event-rule), [事件奖励](../02_Demo数值表.md#event-reward)
  - 验收：
    - 至少4个不同事件
    - 每个事件有2-3个选项
    - 选项有明确后果
    - 风险收益并存

### 事件示例 {#event-examples}

| 事件 | 选项 | 结果 |
|------|------|------|
| 神秘商人 | 购买神秘物品 | -50金币，获得随机遗物 |
| 受伤旅人 | 给予帮助 | -20金币，+1 XP |
| 古老祭坛 | 献上生命 | -10HP，删除1张卡牌 |
| 遗失宝藏 | 拿走金币 | +30金币 |

## 接口契约 {#interface-contract}

### 商店系统输入依赖
- 角色当前金币
- 角色牌库（用于删除）
- 商店商品配置

### 商店系统输出
- 更新后的金币
- 更新后的牌库
- 获得的遗物

### 事件系统输出
- 根据选择改变角色状态

---

## 验收标准 {#acceptance}

### 单元验收
- [ ] 战斗胜利正确获得金币
- [ ] 商店正确显示商品和价格
- [ ] 购买后金币扣除正确
- [ ] 删除卡牌正确从牌库移除
- [ ] 事件选项有实际效果

### 集成验收（关键）
- [ ] **资源循环**：金币获取→商店消费→战力提升
- [ ] **删除策略**：删除基础牌后牌库质量提升
- [ ] **风险收益**：高风险事件有对应高回报

### 体验验收
- [ ] 金币有价值感（不能买光所有东西）
- [ ] 删除卡牌是重要决策
- [ ] 事件选择有趣味性
