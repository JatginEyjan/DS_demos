# DS19 开发拆解与版本规划

## 文档目的

本文档用于指导DS19 Demo的迭代开发，明确各版本的功能边界、交付标准和验收条件。

---

## 一、项目架构概览

### 技术栈建议
- **引擎**: Phaser 3 (沿用DS16技术栈)
- **语言**: TypeScript
- **UI**: 基于Canvas的自定义UI系统
- **数据**: JSON配置文件驱动

### 核心模块划分

```
src/
├── core/                    # 核心系统
│   ├── Game.ts             # 游戏主入口
│   ├── GameState.ts        # 全局状态管理
│   └── EventBus.ts         # 事件总线
├── battle/                  # 战斗系统
│   ├── BattleScene.ts      # 战斗场景
│   ├── TurnManager.ts      # 回合管理
│   ├── CardManager.ts      # 卡牌管理
│   ├── Character.ts        # 角色实体
│   └── Enemy.ts            # 敌人实体
├── map/                     # 地图系统
│   ├── MapScene.ts         # 地图场景
│   ├── Node.ts             # 节点定义
│   └── PathGenerator.ts    # 路径生成
├── character/               # 角色数据
│   ├── CharacterData.ts    # 角色定义
│   ├── CardData.ts         # 卡牌定义
│   └── SkillPool.ts        # 技能池
├── ui/                      # UI组件
│   ├── BattleUI.ts         # 战斗界面
│   ├── MapUI.ts            # 地图界面
│   ├── CardUI.ts           # 卡牌渲染
│   └── DialogUI.ts         # 对话框
├── shop/                    # 商店系统
│   └── ShopScene.ts
└── event/                   # 事件系统
    └── EventScene.ts
```

## 二、版本规划

### Version 0.1 - 核心战斗验证
**目标**: 验证双角色战斗循环是否成立

**功能范围**:
- [x] 单战斗场景（无地图）
- [x] 2名角色（战士+游侠）
- [x] 基础牌库（各10张）
- [x] 1种敌人（标准攻击型）
- [x] 回合制战斗循环
- [x] 角色切换机制
- [x] 能量消耗出牌
- [x] 抽牌/弃牌循环
- [x] 胜负判定

**不包含**:
- 地图系统
- 升级系统
- 商店/事件
- 角色替换

**验收标准**:
1. 可以完成一场完整的双角色战斗
2. 切换角色无卡顿
3. 回合循环正确
4. 牌库洗牌正常

**预计工时**: 3-4天

---

## 四、关键接口定义

### 角色数据接口
```typescript
interface CharacterConfig {
  id: string;
  name: string;
  role: 'warrior' | 'ranger' | 'mage' | 'priest';
  maxHp: number;
  hpGrowth: number;
  baseDeck: CardConfig[];
  skillPool: {
    common: CardConfig[];
    uncommon: CardConfig[];
    rare: CardConfig[];
  };
}
```

### 卡牌数据接口
```typescript
interface CardConfig {
  id: string;
  name: string;
  cost: number;
  type: 'attack' | 'skill' | 'power';
  rarity: 'common' | 'uncommon' | 'rare';
  effects: EffectConfig[];
  description: string;
}
```

### 敌人数据接口
```typescript
interface EnemyConfig {
  id: string;
  name: string;
  hp: number;
  actions: ActionConfig[];
  patterns: PatternConfig[];
}
```

## 五、风险与应对

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|
| 双角色UI信息过载 | 中 | 高 | v0.1快速验证，必要时减抽牌数 |
| 战斗节奏过长 | 中 | 中 | v0.1测试后调整敌人HP |
| 数值平衡困难 | 高 | 中 | 各版本预留调参时间 |
| 切换手感不佳 | 低 | 高 | v0.1重点打磨交互反馈 |
| 内容量过大 | 中 | 中 | 严格按版本范围裁剪 |
