# DS19 Design 3.1 修复版
## 直击灵魂的改进

---

## 一、核心修复：数值链重新计算

### 1.1 敌人等级公式（降低成长速度）
```
enemyLevel = 2 + floor(wave * 1.2) + floor(wave / 5)
```

**重新计算**：
| 波次 | 旧公式 | 新公式 | 玩家等级上限 |
|------|--------|--------|--------------|
| 5 | 约10 | 8 | 12 |
| 10 | 19 | 15 | 19 |
| 20 | 37 | 28 | 34 |
| 30 | 57 | 42 | 49 |

**原则**：敌人等级始终低于玩家等级上限，给玩家成长空间。

### 1.2 玩家等级上限（软上限改为动态匹配）
```
levelCap = 3 + floor(wave * 1.6)
```

**经验效率曲线**：
- 等级 ≤ 上限：100%效率
- 等级 = 上限+1：50%效率
- 等级 = 上限+2：25%效率
- 等级 ≥ 上限+3：10%效率（硬底）

### 1.3 EXP获取与升级需求（完全重算）

**EXP获取公式**：
```
expGain = 2 + floor(enemyLevel / 4)
```

| 敌人等级 | 获得EXP |
|---------|---------|
| 5 | 3 |
| 10 | 4 |
| 15 | 5 |
| 30 | 9 |

**升级需求（固定线性）**：
```
升级所需EXP = 当前等级 × 2
```

**升级曲线验证**：
| 目标等级 | 需要EXP | 累计EXP | 预期波次（按平均5EXP/场）|
|---------|---------|---------|--------------------------|
| 2→3 | 6 | 6 | 第2波 |
| 5→6 | 12 | 42 | 第8波 |
| 10→11 | 22 | 132 | 第26波 |

**关键设计**：Demo 30波，玩家合理等级约12-15级，正好体验完整成长曲线。

## 二、核心修复：牌库与疲劳

### 2.1 初始牌库扩充
```
初始牌库 = 6张打击 + 4张防御 + 2张特性牌 = 12张
```

**每级升级奖励**：选择1张新牌加入牌库（可选移除1张旧牌）
-  DBG的核心乐趣是"精简+优化"，不是"一开始就精简"

### 2.2 疲劳机制调整
- 牌库抽空后，疲劳伤害 = 固定5点（不递增）
- 防止后期疲劳伤害爆炸导致无解

### 2.3 切换与弃牌堆规则（明确）
```
切换精灵时：
1. 手牌保留
2. 弃牌堆保留
3. 抽牌堆 = 新精灵的完整牌库 - 已在弃牌堆的同名牌
```

**防止逃疲劳**：弃牌堆里的牌不会回到新精灵的抽牌堆，疲劳惩罚依然存在。

## 三、核心修复：属性克制链完整定义

### 3.1 8属性完整克制表

```
火 → 草 → 水 → 火（经典三角）
风 → 岩 → 电 → 风（新三角）
光 ↔ 暗（互克）
```

**跨链关系**（中立=1.0x）：
| 属性 | 对光 | 对暗 | 对火 | 对草 | 对水 | 对风 | 对岩 | 对电 |
|------|------|------|------|------|------|------|------|------|
| 光 | - | 2.0x| 1.0x| 1.0x| 1.0x| 1.0x| 1.0x| 1.0x|
| 火 | 1.0x| 1.0x| - | 2.0x| 0.5x| 1.0x| 0.5x| 1.0x|
| 岩 | 1.0x| 1.0x| 2.0x| 1.0x| 1.0x| 0.5x| - | 2.0x|
| 风 | 1.0x| 1.0x| 1.0x| 1.0x| 1.0x| - | 2.0x| 0.5x|

**设计原则**：每个属性有2个克制、2个被克、4个中立，策略空间明确。

## 四、核心修复：战斗流程与胜利条件

### 4.1 战斗流程完整定义
```
1. 遭遇敌人 → 选择出战精灵
2. 战斗开始（玩家先手）
3. 回合循环：
   - 玩家阶段：出牌/使用道具/切换精灵/尝试逃跑
   - 敌人阶段：执行意图
4. 胜利条件：敌人HP归零
5. 结算：获得EXP、金币、可能掉落球
```

### 4.2 捕获流程重定义
```
战斗中选择"捕获" → 消耗当前回合 → 选择精灵球 → 判定捕获

捕获成功：
- 敌人消失（不再替换新敌人）
- 获得该精灵（加入队伍，如果队伍满6只则进入"仓库"）
- 继续战斗或撤退（由玩家选择）

捕获失败：
- 消耗精灵球
- 敌人继续战斗
- 玩家失去本回合行动权（被惩罚）
```

### 4.3 撤退机制
```
战斗中可随时选择"撤退"（消耗当前回合）
撤退成功率 = min(90%, 50% + 己方SPD/敌方SPD × 20%)

撤退成功：
- 结束战斗
- 无EXP、无金币、无掉落
- 进入下一波次选择

撤退失败：
- 浪费一回合
- 敌人正常行动
```

## 五、核心修复：Boss数值与机制

### 5.1 Boss数值重新计算

**基础公式**：
```
Boss等级 = floor(wave × 1.2)
Boss HP = Boss等级 × 10 + 50
Boss ATK = floor(Boss等级 × 1.2)
Boss DEF = floor(Boss等级 × 0.8)
```

**重新计算**：
| Boss | 等级 | HP | ATK | 玩家预期等级 |
|------|------|----|----|--------------|
| 第10波 | 12 | 170 | 14 | 8-10 |
| 第20波 | 24 | 290 | 28 | 12-15 |
| 第30波 | 36 | 410 | 43 | 18-22 |

### 5.2 Boss机制修复

**第20波 Boss：幻影分身（机制明确）**
```
机制：
1. 开场召唤2个分身（分身HP = 本体HP × 25%）
2. 分身攻击力 = 本体攻击力 × 50%
3. 本体在场时，每回合结束时重新召唤缺失的分身
4. 只有击败本体才算胜利

应对策略：
- AOE技能同时清理分身+攻击本体
- 或快速集火本体（分身攻击较低可承受）
```

**第30波 Boss：混沌吞噬者（数值可行）**
```
机制修改：
1. 每3回合（而非每回合）禁用1个属性
2. 禁用持续2回合
3. 被禁用属性伤害 = 0.5x（而非0x），仍可打出但低效
4. 全屏大招每5回合一次，伤害 = BossATK × 1.5

数值验证：
- Boss ATK 43，大招伤害约65
- 玩家精灵HP约150-200，可承受2-3次大招
- 玩家需要预留防御牌或治疗手段
```

## 六、核心修复：糖果与解锁系统

### 6.1 糖果系统逻辑修复
```
原始问题：用开局精灵攒好感解锁...开局精灵？

修复方案：
1. 初始解锁3只starter：焰尾幼龙、水泡龟、叶刃猫
2. 其他9只精灵需要"糖果解锁"才能作为starter
3. 糖果获取：
   - 捕获该精灵时 +50好感度
   - 战斗中该精灵濒死时 +10好感度（安慰奖）
   - 每场战斗该精灵存活 +3好感度
4. 好感度达到阈值 → 获得1颗该精灵的糖果
5. 收集3颗糖果 → 解锁该精灵作为starter
```

### 6.2 好感度阈值调整（Demo可行）
| 稀有度 | 解锁需糖果数 | 单局可获得好感度（估算）|
|--------|-------------|------------------------|
| 普通 | 3颗 | 约60（捕获50+存活3场9）|
| 稀有 | 3颗 | 约60 |
| 史诗 | 2颗 | 约50（捕获更难）|
| 传说 | 2颗 | 约50（Boss掉落捕获）|

**设计意图**：Demo 30波可解锁2-3只新starter，形成"再玩一局试试新精灵"的动力。

## 七、核心修复：经济与回复

### 7.1 金币获取调整
| 来源 | 旧数值 | 新数值 |
|------|--------|--------|
| 普通敌人 | 10+波 | 15+波 |
| 精英敌人 | 30+波×2 | 40+波×2 |
| Boss | 100+波×5 | 80+波×3 |

### 7.2 商店与回复
```
商店触发：每5波后出现
商店内容：
- 普通球：40金币（降价）
- 高级球：120金币
- 伤药（回30HP）：30金币
- 大伤药（回60HP）：60金币
- 移除卡牌：50金币

篝火出现：每10波后（Boss战后）
篝火选项（3选1）：
1. 回复全队30%HP
2. 复活1只濒死精灵（50%HP）
3. 移除2张卡牌
```

## 九、修复总结：从3.0到3.1的核心变更

| 问题 | 3.0设计 | 3.1修复 |
|------|---------|---------|
| 敌人等级碾压 | 57级vs玩家20级 | 42级vs玩家22级 |
| EXP跟不上 | 升级需求递增，获取固定 | 固定2×等级需求，获取随敌人成长 |
| 牌库太小 | 8张 | 12张+升级可精简 |
| 疲劳爆炸 | 递增1,2,3... | 固定5点 |
| 属性链缺失 | 风/岩/电未定义 | 完整8属性克制表 |
| 捕获自杀 | 捕获后打新敌人 | 捕获=敌人消失 |
| 糖果逻辑矛盾 | 解锁已解锁 | 明确初始3只，其他需解锁 |
| Boss打不死 | 数值鸿沟 | 重新计算后5-6回合可击败 |
| 无回复手段 | 商店每5波，伤药贵 | 商店降价，篝火每10波 |

## 十、仍然需要验证的问题

1. **12张牌库在30波中是否足够**？需要实际测试疲劳节奏。
2. **3选1无保底**是否依然劝退？考虑加入"至少1张稀有"规则。
3. **濒死不可复活**是否过于硬核？考虑篝火可复活但降低HP回复。
4. **6只精灵上限**在后期是否够用？可能需要在商店增加"扩容"选项。
