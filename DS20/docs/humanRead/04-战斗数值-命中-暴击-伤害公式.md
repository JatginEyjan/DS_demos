# 战斗数值、命中、暴击与伤害公式

## 1. 永久数值生成

来源：

- `src/field/pokemon.ts`
- `src/data/nature.ts`
- `src/utils/common.ts`

## 1.1 IV 生成

默认没有 dataSource 时：

```text
id = randSeedInt(4294967296)
ivs = getIvsFromId(id)
```

`getIvsFromId(id)` 直接把 32 位整数切成 6 个 5-bit：

```text
HP    = bits 25..29
ATK   = bits 20..24
DEF   = bits 15..19
SPATK = bits 10..14
SPDEF = bits 5..9
SPD   = bits 0..4
```

每项范围 `0~31`。

## 1.2 基础能力值

`Pokemon.calculateBaseStats()`

基础流程：

1. 复制种族 `baseStats`
2. 应用挑战 `FLIP_STAT`
3. 应用 `PokemonBaseStatTotalModifier`
4. 应用 `PokemonBaseStatFlatModifier`
5. 若是融合：
   - 与融合对象 baseStats 做逐项平均并向上取整
6. 若是 `SplicedOnly` 但当前未融合：
   - 所有 baseStats 直接减半并向上取整
7. 应用 `BaseStatModifier`

## 1.3 最终永久能力公式

`Pokemon.calculateStats()`

### HP

```text
base = floor((2 * baseStat + iv) * level * 0.01)
hp = base + level + 10
```

然后继续应用：

- `PokemonIncrementingStatModifier`
- 若拥有 `Wonder Guard`，HP 直接设为 `1`

### 非 HP

```text
stat = floor((2 * baseStat + iv) * level * 0.01) + 5
```

然后乘以性格修正：

- 增益性格：`1.1`
- 减益性格：`0.9`
- 中性：`1.0`

实现细节：

- 性格倍率大于 `1` 用 `ceil`
- 性格倍率小于 `1` 用 `floor`
- 最终至少为 `1`

之后再应用：

- `PokemonIncrementingStatModifier`

## 2. 性格倍率

`getNatureStatMultiplier(nature, stat)`

单项只会出现三种结果：

- `1.1`
- `0.9`
- `1`

系统只会修正：

- `ATK`
- `DEF`
- `SPATK`
- `SPDEF`
- `SPD`

`HP` 不受性格影响。

## 3. 有效能力值

`Pokemon.getEffectiveStat(...)`

计算顺序：

1. 从永久能力取值
2. 应用持有物 `StatBoosterModifier`
3. 遍历场上单位，应用 `FieldMultiplyStatAbAttr`
4. 应用自身 `StatMultiplierAbAttr`
5. 应用友方 `AllyStatMultiplierAbAttr`
6. 乘以能力阶段倍率 `getStatStageMultiplier`
7. 根据能力种类叠加特殊规则

### 3.1 特殊规则

#### ATK

- 有 `SLOW_START` 标签时减半

#### DEF

- 冰系且天气为 `SNOW` 时乘 `1.5`

#### SPDEF

- 岩石系且天气为 `SANDSTORM` 时乘 `1.5`

#### SPD

- 本方场地有 `TAILWIND` 时乘 `2`
- 本方场地有 `GRASS_WATER_PLEDGE` 时减半

最终值：

```text
max(floor(ret), 1)
```

## 4. 能力阶段倍率

`Pokemon.getStatStageMultiplier(...)`

### 4.1 基础公式

```text
multiplier = max(2, 2 + stage) / max(2, 2 - stage)
```

最终上限为 `4`。

### 4.2 暴击对能力阶段的影响

发生暴击时：

- 攻击方的 `ATK / SPATK` 负能力阶段会被抹到 `0`
- 防守方的 `DEF / SPDEF` 正能力阶段会被抹到 `0`

### 4.3 忽略能力阶段

以下逻辑会让阶段倍率直接变成 `1`：

- 对手的 `IgnoreOpponentStatStagesAbAttr`
- 招式的 `IgnoreOpponentStatStagesAttr`

### 4.4 临时强化

若不忽略持有物：

- 会继续应用 `TempStatStageBoosterModifier`

## 5. 命中倍率

`Pokemon.getAccuracyMultiplier(target, sourceMove)`

### 5.1 一击必杀招式

若招式带 `OneHitKOAccuracyAttr`：

- 直接返回 `1`

### 5.2 ACC / EVA 公式

取：

- 攻击方 `ACC stage`
- 防守方 `EVA stage`

若二者不相等：

```text
if userAcc > targetEva:
  multiplier = (3 + min(userAcc - targetEva, 6)) / 3
else:
  multiplier = 3 / (3 + min(targetEva - userAcc, 6))
```

再应用：

- 攻击方 `StatMultiplierAbAttr(ACC)`
- 防守方 `StatMultiplierAbAttr(EVA)`
- 友方 `AllyStatMultiplierAbAttr`

最终：

```text
accuracyMultiplier / evasionMultiplier
```

## 6. 暴击

### 6.1 暴击阶段

`Pokemon.getCritStage(source, move)`

组成来源：

- 招式 `HighCritAttr`
- `CritBoosterModifier`
- `TempCritBoosterModifier`
- 能力 `BonusCritAbAttr`
- `CritBoostTag`

### 6.2 暴击概率表

`Pokemon.getCriticalHitResult()`

把暴击阶段 clamp 到 `0~3` 后，直接查表：

- `0` 阶：`1/24`
- `1` 阶：`1/8`
- `2` 阶：`1/2`
- `3+` 阶：必暴

### 6.3 暴击倍率

默认：

```text
criticalMultiplier = 1.5
```

再通过 `MultCritAbAttr` 修改。

### 6.4 暴击阻止

即使前面判定已暴击，仍会被以下效果取消：

- `BlockCritAbAttr`
- arena `NoCritTag`

## 7. STAB

`Pokemon.calculateStabMultiplier(source, move, ignoreSourceAbility, simulated)`

基础规则：

- 无属性招式 `TypelessAttr` 不吃 STAB
- 招式属性与自身属性匹配：`+0.5`
- 应用 `CombinedPledgeStabBoostAttr`
- 应用能力 `StabBoostAbAttr`
- 太晶后，若太晶属性等于招式属性且不是 `STELLAR`：再 `+0.5`
- `STELLAR` 太晶：
  - 若招式属性原本在自身属性里：`+0.5`
  - 否则：`+0.2`

最终上限：

```text
STAB <= 2.25
```

## 8. 基础伤害公式

`Pokemon.getBaseDamage(...)`

```text
levelMultiplier = (2 * source.level) / 5 + 2
power = move.calculateBattlePower(...)
sourceAtk = 攻方有效攻击或特攻
targetDef = 守方有效防御或特防

baseDamage = (levelMultiplier * power * sourceAtk) / targetDef / 50 + 2
```

## 9. 总伤害公式

`Pokemon.getAttackDamage(...)`

### 9.1 固定伤害与一击必杀

优先级高于常规伤害：

- `FixedDamageAttr`：直接返回固定伤害
- `OneHitKOAttr`：直接返回 `this.hp`

### 9.2 总公式

```text
damage =
baseDamage
* targetMultiplier
* multiStrikeEnhancementMultiplier
* arenaAttackTypeMultiplier
* glaiveRushMultiplier
* criticalMultiplier
* randomMultiplier
* stabMultiplier
* typeMultiplier
* burnMultiplier
* screenMultiplier
* hitsTagMultiplier
* mistyTerrainMultiplier
```

最后：

```text
toDmgValue(...)
```

之后还会继续被能力、敌方 token、招式 attr 再修正。

### 9.3 各乘区

#### targetMultiplier

命中多个未倒下目标时：

- `0.75`

否则：

- `1`

#### multiStrikeEnhancementMultiplier

来自：

- `PokemonMultiHitModifier`

用于 `Multi-Lens`、`Parental Bond` 等额外打击增强。

#### arenaAttackTypeMultiplier

来自：

- `globalScene.arena.getAttackTypeMultiplier(moveType, source.isGrounded())`

包含天气、场地、地面接触等环境对属性伤害的修正。

#### glaiveRushMultiplier

如果目标有 `RECEIVE_DOUBLE_DAMAGE` 标签：

- `2`

否则：

- `1`

#### randomMultiplier

- 模拟计算：`1`
- 实战计算：`randBattleSeedIntRange(85, 100) / 100`

即 `0.85~1.00`

#### burnMultiplier

满足以下条件时：

- 物理招式
- 攻击方处于 `BURN`
- 招式没有 `BypassBurnDamageReductionAttr`
- 且能力没有 `BypassBurnDamageReductionAbAttr`

则：

- `0.5`

#### screenMultiplier

防守方对应侧有屏障时由 `WeakenMoveScreenTag` 计算。
暴击时直接跳过这一乘区。

#### hitsTagMultiplier

若招式具有 `HitsTagAttr(doubleDamage = true)`，
且目标正带对应 `BattlerTagType`：

- 每个命中的 tag 乘 `2`

#### mistyTerrainMultiplier

当以下条件同时满足：

- 当前地形是 `MISTY`
- 目标接地
- 招式属性是 `DRAGON`

则：

- `0.5`

## 10. 伤害后修正

总伤害生成后继续应用：

1. 攻击方能力 `MoveDamageBoostAbAttr`
2. 若攻击者是敌人：`EnemyDamageBoosterModifier`
3. 若防守者是敌人：`EnemyDamageReducerModifier`
4. 防守方能力 `ReceivedMoveDamageMultiplierAbAttr`
5. 双打中若队友有效在场：`AlliedFieldDamageReductionAbAttr`
6. 招式 `ModifiedDamageAttr`
7. 满血时的 `PreDefendFullHpEndureAbAttr`

## 11. 属性克制结果

`HitResult` 只看 `typeMultiplier`：

- `< 1`：`NOT_VERY_EFFECTIVE`
- `> 1`：`SUPER_EFFECTIVE`
- `= 1`：`EFFECTIVE`

如果属性倍率乘环境倍率等于 `0`：

- 返回 `NO_EFFECT`
- `Sheer Cold` 特判成 `IMMUNE`
