## 1.1 击败单位基础经验

`Pokemon.getExpValue()`

```text
expValue = (speciesBaseExp * level) / 5 + 1
```

源码注释明确说明：

- 已移除“胜者等级参与经验公式”的逻辑
- 目的是不让玩家围绕经验最大化操作

## 1.2 队伍分配入口

`BattleScene.applyPartyExp(expValue, pokemonDefeated, useWaveIndexMultiplier = false, participantIds)`

### 1.2.1 波次倍率

当 `useWaveIndexMultiplier = true`：

```text
expValue = floor((expValue * currentWave) / 5 + 1)
```

### 1.2.2 战斗类型修正

- 训练家战：`expValue *= 1.5`
- 神秘事件战：`expValue *= mysteryEncounter.expMultiplier`

## 2. 参与分配规则

### 2.1 参与者定义

`participantIds` 是本战共享经验的玩家单位 PID 集合。

在 `TurnInitPhase` 中，所有 active 的玩家单位都会 `addParticipant()`。

### 2.2 参与单位经验倍率

对于一个参与过击杀分配的单位：

```text
expMultiplier += 1 / participantIds.size
```

如果参与人数大于 1 且持有 `MultipleParticipantExpBonusModifier`：

```text
expMultiplier += stackCount * 0.2
```

### 2.3 未参与但有 Exp Share

```text
expMultiplier += (expShareStackCount * 0.2) / participantIds.size
```

### 2.4 Pokerus

若单位有 Pokerus：

- 经验倍率再乘 `1.5`

### 2.5 最终个人经验

```text
pokemonExp = floor(expValue * expMultiplier)
```

然后继续应用 `PokemonExpBoosterModifier`。

## 3. Exp Balance

若玩家拥有 `ExpBalanceModifier`：

1. 统计所有能吃经验成员的总等级与总经验
2. 计算：

```text
medianLevel = floor(totalLevel / expPartyMembers.length)
```

3. 把等级 `<= medianLevel` 的成员视为接受平衡经验目标
4. 计算：

```text
splitExp = floor(totalExp / recipientCount)
```

5. 对每个成员：

```text
newExp = Linear(originalExp, targetExp, 0.2 * stackCount)
```

其中：

- 低等级成员 targetExp = `splitExp`
- 其余成员 targetExp = `0`

## 4. 等级曲线

`getLevelTotalExp(level, growthRate)`

### 4.1 1~99 级

直接读 `expLevels[growthRate][level - 1]`，但除 `MEDIUM_FAST` 外还会混入中快曲线：

```text
ret = floor(levelExp * 0.325 + getLevelTotalExp(level, MEDIUM_FAST) * 0.675)
```

也就是：

- `MEDIUM_FAST` 用原表
- 其他成长率是“原表 32.5% + 中快 67.5%”

### 4.2 100 级以上

直接使用公式，再继续做同样的 `32.5% + 67.5%` 混合。

原始公式：

- `ERRATIC`: `(level^4 + level^3 * 2000) / 3500`
- `FAST`: `(level^3 * 4) / 5`
- `MEDIUM_FAST`: `level^3`
- `MEDIUM_SLOW`: `(level^3 * 6) / 5 - 15 * level^2 + 100 * level - 140`
- `SLOW`: `(level^3 * 5) / 4`
- `FLUCTUATING`: `(level^3 * (level / 2 + 8) * 4) / (100 + level)`

### 4.3 当前等级内经验

```text
levelExp = totalExp - getLevelTotalExp(level, growthRate)
```

## 5. 等级上限

`BattleScene.getMaxExpLevel()`

```text
waveIndex = ceil(currentWave / 10) * 10
difficultyWaveIndex = gameMode.getWaveForDifficulty(waveIndex)
baseLevel = (1 + difficultyWaveIndex / 2 + (difficultyWaveIndex / 25)^2) * 1.2
maxExpLevel = ceil(baseLevel / 2) * 2 + 2
```

## 6. 好感

来源：

- `src/data/balance/starters.ts`
- `src/constants.ts`
- `src/field/pokemon.ts`

### 6.1 基础常量

- 单次战斗获得好感：`3`
- 使用 Rare Candy 获得好感：`6`
- 阵亡损失好感：`5`
- Rare Candy 好感上限：`200`
- 训练家敌人最小好感：`50`
- 训练家敌人满好感波次：`145`

### 6.2 addFriendship

`Pokemon.addFriendship(friendship, capped = false)`

若 `friendship <= 0`：

- 直接修改 `this.friendship`
- 不影响糖果进度

若 `friendship > 0`：

1. 先应用 `PokemonFriendshipBoosterModifier`
2. 计算新好感
3. 若 `capped = true` 且超过 `RARE_CANDY_FRIENDSHIP_CAP = 200`
   - 本次增长最多到 `200`
4. 最终实际好感仍会被整体 clamp 到 `255`

### 6.3 糖果进度

正向好感会转成 starterData 的 `friendship` 进度。

倍率：

- 经典模式：`timedEventManager.getClassicFriendshipMultiplier()`，默认基础是 `3`
- 非经典：`1`

融合时：

- 如果 event 提升融合收益，除以 `1.5`
- 否则除以 `2`

实际用于糖果的量：

```text
candyFriendshipAmount = floor(friendship * candyFriendshipMultiplier)
```

达到对应 starter 成本的 friendship cap 后就发糖果。

## 7. 捕捉公式

`AttemptCapturePhase.start()`

### 7.1 球倍率

`getPokeballCatchMultiplier(type)`

- `POKEBALL = 1`
- `GREAT_BALL = 1.5`
- `ULTRA_BALL = 2`
- `ROGUE_BALL = 3`
- `MASTER_BALL = -1`
- `LUXURY_BALL = 1`

其中 `MASTER_BALL = -1` 在后续流程中被视为直接成功。

### 7.2 状态倍率

`getStatusEffectCatchRateMultiplier(status)`

- `POISON / TOXIC / PARALYSIS / BURN = 1.5`
- `SLEEP / FREEZE = 2.5`
- 无状态 = `1`

### 7.3 闪光倍率

`SHINY_CATCH_RATE_MULTIPLIER = 2`

实际运行时：

- 若敌方是闪光，取 `timedEventManager.getShinyCatchMultiplier()`
- 默认事件常量对应倍数为 `2`

### 7.4 修正捕获率

```text
_3m = 3 * maxHp
_2h = 2 * currentHp
catchRate = species.catchRate

modifiedCatchRate =
round(((_3m - _2h) * catchRate * ballMultiplier / _3m) * statusMultiplier * shinyMultiplier)
```

### 7.5 摇动概率

```text
shakeProbability = round(65536 / (255 / modifiedCatchRate)^0.1875)
```

源码注释注明该公式取自 Gen 6。

### 7.6 critical capture 概率

`getCriticalCaptureChance(modifiedCatchRate)`

若是 `Fresh Start Challenge`：

- 直接为 `0`

否则：

```text
dexCount = 已捕获图鉴数
dexMultiplier =
  Daily 或 dexCount > 800 -> 2.5
  dexCount > 600 -> 2
  dexCount > 400 -> 1.5
  dexCount > 200 -> 1
  dexCount > 100 -> 0.5
  其他 -> 0

criticalChance = floor(catchingCharmMultiplier * dexMultiplier * min(255, modifiedCatchRate) / 6)
```

### 7.7 摇球判定

先判定是否 critical capture：

```text
randBattleSeedInt(256) < criticalCaptureChance
```

然后进入摇球。

普通捕捉：

- 共做 `3` 次摇动检查

critical capture：

- 动画上只做 `1` 次有效摇动检查

每次摇动通过条件：

- `ballMultiplier === -1`，即大师球
- 或 `isCritical`
- 或 `modifiedCatchRate >= 255`
- 或 `randBattleSeedInt(65536) < shakeProbability`

全部通过则成功。

## 8. 捕捉成功后的处理

`AttemptCapturePhase.catch()`

会做的事：

- 验证隐藏特性成就
- 验证次神 / 神兽捕捉成就
- `addToParty(pokeballType, slotIndex)`
- 记录图鉴已见 / 已捕
- 设置捕获来源球种

## 9. 进化

核心进化条件数据来自：

- `src/data/balance/pokemon-evolutions.ts`

系统在战斗与成长 phase 中根据条件检查。

文档中与核心数值直接相关的条件类型包括：

- 等级
- 好感
- 招式
- 持有物
- 时段
- 性别
- 形态
- 特定环境

好感条件在源码里直接比较：

```text
pokemon.friendship >= cond.value
```
