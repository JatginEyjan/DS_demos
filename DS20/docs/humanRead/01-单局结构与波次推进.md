# 单局结构与波次推进

## 1. 入口参数

来源：

- `src/game-mode.ts`
- `src/battle-scene.ts`
- `src/battle.ts`

单局起始参数由 `GameMode` 提供。

### 1.1 起始等级

`GameMode.getStartingLevel()`

- `Overrides.STARTING_LEVEL_OVERRIDE > 0` 时直接使用 override。
- `Daily` 固定为 `20`。
- 其他模式固定为 `5`。

### 1.2 起始金钱

`GameMode.getStartingMoney()`

- `Overrides.STARTING_MONEY_OVERRIDE > 0` 时使用 override。
- `Daily` 先读 `getDailyStartingMoney()`。
- 其余情况默认 `1000`。

### 1.3 起始地形

`GameMode.getStartingBiome()`

- `Overrides.STARTING_BIOME_OVERRIDE` 存在时使用 override。
- `Daily` 使用 `getDailyStartingBiome()`。
- 其余模式默认 `BiomeId.TOWN`。

## 2. 新波次创建

`BattleScene.newBattle()`

执行顺序：

1. 调用 `getNewBattleProps()` 计算初始波次参数。
2. 用 `resetSeed(waveIndex)` 重置当前波次随机种子。
3. 根据来源决定本波配置：
   - 从存档读取：`handleSavedBattle`
   - 固定战：`handleFixedBattle`
   - 普通生成：`handleNonFixedBattle`
4. 调用 `checkIsDouble()` 计算是否双打。
5. 创建新的 `Battle` 实例。
6. `currentBattle.incrementTurn()`，初始 turn 直接加到 `1`。
7. 如果不是从 session 波次恢复，则对上一波执行 `doPostBattleCleanup()`。

### 2.1 默认波次号

`getNewBattleProps(fromSession == null)`

- 如果当前已有 battle，则 `waveIndex = currentBattle.waveIndex + 1`
- 否则使用 `Overrides.STARTING_WAVE_OVERRIDE ?? STARTING_WAVE`

## 3. Battle 对象初始化

`Battle.constructor()`

关键字段：

- `waveIndex`
- `battleType`
- `trainer`
- `double`
- `battleSpec`
- `enemyLevels`
- `enemyParty`
- `playerParticipantIds`
- `escapeAttempts`
- `postBattleLoot`

### 3.1 BattleSpec

`Battle.initBattleSpec()`

- 默认 `BattleSpec.DEFAULT`
- 如果 `isWaveFinal(waveIndex)` 且模式是 `Classic`，则设为 `BattleSpec.FINAL_BOSS`

### 3.2 敌方等级数组

`Battle.constructor()`

- 训练家战：`trainer?.getPartyLevels(this.waveIndex)`
- 野生战：
  - 单打创建 `1` 个等级
  - 双打创建 `2` 个等级
  - 每个等级都由 `getLevelForWave()` 独立生成

## 4. 波次等级公式

来源：

- `src/battle.ts`
- `src/game-mode.ts`

### 4.1 难度波次

`GameMode.getWaveForDifficulty(waveIndex, ignoreCurveChanges = false)`

- `Daily`：`waveIndex + 30 + floor(waveIndex / 5)`，除非 `ignoreCurveChanges = true`
- 其他模式：直接等于 `waveIndex`

### 4.2 基础等级曲线

`Battle.getLevelForWave()`

```text
levelWaveIndex = gameMode.getWaveForDifficulty(waveIndex)
baseLevel = 1 + levelWaveIndex / 2 + (levelWaveIndex / 25)^2
bossMultiplier = 1.2
```

### 4.3 Boss 波次等级

当 `gameMode.isBoss(waveIndex)` 为真：

```text
ret = floor(baseLevel * 1.2)
```

分支：

- 如果是经典最终战 `BattleSpec.FINAL_BOSS`，或 `waveIndex % 250 === 0`
  - 返回 `ceil(ret / 25) * 25`
- 否则
  - 如果不是最终波，额外加入：

```text
levelOffset = round(random(-1, 1) * floor(levelWaveIndex / 10))
finalLevel = ret + levelOffset
```

### 4.4 非 Boss 波次等级

```text
deviation = 10 / levelWaveIndex
levelOffset = abs(randSeedGaussForLevel(deviation))
finalLevel = max(round(baseLevel + levelOffset), 1)
```

`randSeedGaussForLevel(value)` 的实现不是标准高斯分布，而是：

```text
rand = 0
重复 value 次:
  rand += randSeedFloat()
return rand / value
```

即把若干次 `0~1` 的均匀随机求平均，再取绝对值。

## 5. 波次类型判定

来源：

- `src/game-mode.ts`
- `src/battle-scene.ts`

### 5.1 最终波

`GameMode.isWaveFinal(waveIndex)`

- `Classic / Challenge`: `waveIndex === 200`
- `Endless / Spliced Endless`: `waveIndex % 250 === 0`
- `Daily`: `waveIndex === 50`

### 5.2 Boss 波

`GameMode.isBoss(waveIndex)`

- 每 `10` 波一次：`waveIndex % 10 === 0`

### 5.3 Endless 特殊 Boss

- `isEndlessBoss(waveIndex)`：`waveIndex % 50 === 0`
- `isEndlessMinorBoss(waveIndex)`：`waveIndex % 250 === 0`
- `isEndlessMajorBoss(waveIndex)`：`waveIndex % 1000 === 0`

## 6. 新 biome 判定

`BattleScene.isNewBiome(currentBattle)`

满足以下任一条件即视为进入新 biome：

- `waveIndex % 10 === 0`
- Endless 短 biome 模式下，`waveIndex % 5 === 0`
- Endless 或 Daily 下，`waveIndex % 50 === 49`

也就是说：

- 普通经典：每 10 波切 biome
- 短 biome 模式：每 5 波切 biome
- Endless / Daily 在 49 波这种切换前一波也会触发生物群系切换逻辑

## 7. 双打判定

`BattleScene.checkIsDouble()`

强制不允许双打的情况：

- `isWaveFinal(waveIndex)`
- `isEndlessBoss(waveIndex)`
- `battleType === MYSTERY_ENCOUNTER`

### 7.1 override

`doCheckDoubleOverride(waveIndex)`

- `"double"`：恒为双打
- `"single"`：恒为单打
- `"even-doubles"`：偶数波双打
- `"odd-doubles"`：奇数波双打

### 7.2 普通双打概率

`getDoubleBattleChance(newWaveIndex)`

初始分母：

- Boss 波：`32`
- 其他波：`8`

最终概率：

```text
是否双打 = randSeedInt(doubleChance) === 0
```

即：

- Boss 波基础概率 `1/32`
- 非 Boss 波基础概率 `1/8`

然后会继续叠加：

- `DoubleBattleChanceBoosterModifier`
- 场上宝可梦能力 `DoubleBattleChanceAbAttr`

最终最小分母不低于 `1`。

## 8. 后处理与跨波保留

`BattleScene.doPostBattleCleanup(lastBattle, maxExpLevel)`

### 8.1 需要重置 arena 的情况

`resetArenaState = true` 的条件：

- 进入新 biome
- 当前战斗是训练家战
- 当前战斗是神秘事件战
- 当前战斗是最终 Boss 规格

### 8.2 重置内容

当 `resetArenaState = true`：

- `arena.resetArenaEffects()`
  - 清天气
  - 清地形
  - 清 arena tags
  - 清 arena 内玩家阵亡计数
- 玩家当前上场位执行 `ReturnPhase`
- 全队执行：
  - `resetBattleAndWaveData()`
  - `resetTera()`
  - `PostBattleInitAbAttr`
- `Terapagos` 额外把 `arena.playerTerasUsed = 0`

### 8.3 非新 biome 的中间刷新

如果不是新 biome，且 `waveIndex % 10 === 5`：

- 执行 `arena.updatePoolsForTimeOfDay()`

即同一 biome 中间会刷新一次按昼夜切换后的野生池。

### 8.4 波次结束后推进

- 如果模式没有随机 biome 且不是新 biome：
  - 推入 `NextEncounterPhase`
- 否则：
  - 推入 `NewBiomeEncounterPhase`

## 9. 等级上限

`BattleScene.getMaxExpLevel(ignoreLevelCap = false)`

```text
waveIndex = ceil(currentWave / 10) * 10
difficultyWaveIndex = gameMode.getWaveForDifficulty(waveIndex)
baseLevel = (1 + difficultyWaveIndex / 2 + (difficultyWaveIndex / 25)^2) * 1.2
maxExpLevel = ceil(baseLevel / 2) * 2 + 2
```

特殊情况：

- `Overrides.LEVEL_CAP_OVERRIDE > 0`：直接返回 override
- `ignoreLevelCap = true` 或 override 为负：返回 `Number.MAX_SAFE_INTEGER`

## 10. 波次结束后的 biome 选择前置

`SelectBiomePhase.setNextBiomeAndEnd()`

当 `nextWaveIndex % 10 === 1` 时，切 biome 前固定执行：

1. `MoneyInterestModifier`
2. 若挑战不禁用队伍恢复，则 `PartyHealPhase`
3. 否则直接进入 `SelectModifierPhase`
4. 再执行 `SwitchBiomePhase(nextBiome)`

这意味着一个标准 10 波循环的切换点是：

- 战斗结束
- 利息结算
- 队伍恢复或奖励
- 切 biome
- 进入下一波
