# 地图、遭遇与敌人生成规则

## 1. biome 进入与分支

来源：

- `src/phases/select-biome-phase.ts`
- `src/battle-scene.ts`
- `src/field/arena.ts`

### 1.1 直接进入 END 的条件

`SelectBiomePhase.start()`

满足以下条件之一，下一 biome 直接设为 `BiomeId.END`：

- 经典模式下，`nextWaveIndex + 9` 是最终波
- Daily 模式下，`nextWaveIndex` 是最终波
- 短 biome 模式下，`nextWaveIndex % 50 === 0`

### 1.2 随机 biome 模式

`gameMode.hasRandomBiomes = true` 时：

- 直接调用 `generateNextBiome(nextWaveIndex)`
- `waveIndex % 50 === 0` 时固定为 `END`
- 否则调用 `BattleScene.generateRandomBiome(waveIndex)`

### 1.3 固定链接 biome 模式

从当前 biome 的 `biomeLinks` 读取下一 biome。

- 如果 `biomeLinks.length > 1`
  - 先过滤掉带权重条件但随机失败的链接
  - 如果过滤后仍大于 1 且玩家持有 `MapModifier`
    - 进入手动选择 UI
  - 否则在剩余 biome 中随机选一个
- 如果 `biomeLinks.length === 1`
  - 直接进入该 biome
- 如果没有链接
  - fallback 为随机 biome

## 2. 随机 biome 权重公式

`BattleScene.generateRandomBiome(waveIndex)`

```text
relWave = waveIndex % 250
biomes = 所有 biome，排除 TOWN 和 END
maxDepth = biomeDepths[END][0] - 2
```

对每个深度 `i` 计算：

```text
depthWeight[i] =
((1 - min(abs(i / (maxDepth - 1) - relWave / 250) + 0.25, 1)) / 0.75) * 250
```

对每个 biome：

```text
weight = ceil(depthWeight[biomeDepths[biome][0] - 1] / biomeDepths[biome][1])
```

最后做累计阈值抽取。

这个过程完全由 biome 深度表 `biomeDepths` 控制。

## 3. 时间、天气、地形

### 3.1 时间刷新

`Arena.updatePoolsForTimeOfDay()`

- 根据 `getTimeOfDay()` 重新合并 biome 的宝可梦池
- 每个 tier 的实际池为：

```text
pokemonPool[tier] = 原始池[TimeOfDay.ALL] + 原始池[当前时段]
```

- 如果时段未变化，不重建池

### 3.2 biome 天气

`Arena.setBiomeWeather()`

步骤：

1. 读取 biome 自带 `weatherPool`
2. 若当前有 timed event，且该 biome 存在事件天气覆盖，则改用事件池
3. 夜晚或黄昏时：
   - `SUNNY` 权重强制设为 `0`
   - 如果 biome 是 `FOREST`，`FOG` 权重设为 `1`
4. 如果所有天气权重都为 `0`
   - 强制 `NONE = 1`
5. 调用 `weightedPick(weatherMap)` 选天气

### 3.3 biome 地形

`Arena.setBiomeTerrain()`

- 若存在 `Overrides.STARTING_TERRAIN_OVERRIDE`，直接覆盖
- 否则读取 biome 的 `terrainPool`
- 将所有 `TerrainType` 补齐为 map 后执行 `weightedPick`

## 4. 训练家波次判定

来源：

- `src/game-mode.ts`
- `src/field/arena.ts`
- `src/battle-scene.ts`

### 4.1 Daily

`GameMode.isWaveTrainer(waveIndex)`

优先读 `getDailyTrainerManipulation(waveIndex)`。

如果没有 daily override：

- `waveIndex % 10 === 5` 时是训练家
- `waveIndex % 10 === 0 && waveIndex > 10 && !isWaveFinal(waveIndex)` 时也是训练家

即 Daily 默认训练家层：

- 5, 15, 20, 25, 30, 35, 40, 45

### 4.2 非 Daily

固定 gym/大训练家层：

- `waveIndex % 30 === (offsetGym ? 0 : 20)`
- 且不是最终波

普通训练家候选层：

- `waveIndex % 10 !== 1`
- `waveIndex % 10 !== 0`
- 也就是不会在 `X0` 和 `X1` 做普通训练家随机

普通训练家还要满足：

- biome 有 `trainerChance`
- 与相邻固定战或 gym 层不冲突
- 向前 2 波到向后 2 波范围内不碰撞

最终随机：

```text
allowTrainerBattle && trainerChance && randSeedInt(trainerChance) === 0
```

`trainerChance` 是分母。
例如 `trainerChance = 4`，概率就是 `1/4`。

## 5. 训练家类型抽取

`Arena.randomTrainerType(waveIndex, isBoss = false)`

### 5.1 稀有度池

Boss 训练家使用 `0~63` 抽取，普通训练家使用 `0~511` 抽取。

#### Boss 训练家 tier

`generateBossBiomeTier(tierValue)`

- `20~63`：`BOSS`，44/64
- `6~19`：`BOSS_RARE`，14/64
- `1~5`：`BOSS_SUPER_RARE`，5/64
- `0`：`BOSS_ULTRA_RARE`，1/64

#### 普通训练家 tier

`generateNonBossBiomeTier(tierValue)`

- `156~511`：`COMMON`，356/512
- `32~155`：`UNCOMMON`，124/512
- `6~31`：`RARE`，26/512
- `1~5`：`SUPER_RARE`，5/512
- `0`：`ULTRA_RARE`，1/512

### 5.2 空池降级

若当前 tier 池为空：

- 循环 `tier--`
- 一直降到该 biome 的 trainerPool 中找到非空 tier

### 5.3 默认回退

所有 tier 都空时，返回 `TrainerType.BREEDER`。

## 6. 野生宝可梦生成

`Arena.randomSpecies(waveIndex, level, attempt = 0, luckValue = 0, isBoss?)`

### 6.1 override 优先级

先检查 `gameMode.getOverrideSpecies(waveIndex)`：

- Daily 最终 Boss 可被自定义 species 覆盖
- Daily 强制波次也可覆盖 species

有 override 时直接返回，不走 biome 池。

### 6.2 是否进入 Boss 池

`isBossSpecies` 成立条件：

- `getEncounterBossSegments(waveIndex, level) > 0`
- 且当前 biome 的 `BiomePoolTier.BOSS` 非空
- 且以下条件之一成立：
  - biome 不是 `END`
  - 模式是 `Classic`
  - 该波是最终波

### 6.3 稀有度 tier 抽法

若 `forcedTier = getDailyForcedWaveBiomePoolTier(waveIndex)` 不为空，直接用它。

否则：

- Boss 池：`rollMax = 64`
- 普通池：`rollMax = 512`

幸运值修正：

```text
luckModifier = luckValue * 0.5    // Boss
luckModifier = luckValue * 2      // 非 Boss
rngRoll = randSeedInt(rollMax - luckModifier)
```

也就是幸运值会直接缩小随机上限，使高稀有 tier 更容易出现。

### 6.4 稀有度概率

#### 普通野生池

- `156~511`：`COMMON`
- `32~155`：`UNCOMMON`
- `6~31`：`RARE`
- `1~5`：`SUPER_RARE`
- `0`：`ULTRA_RARE`

#### Boss 野生池

- `20~63`：`BOSS`
- `6~19`：`BOSS_RARE`
- `1~5`：`BOSS_SUPER_RARE`
- `0`：`BOSS_ULTRA_RARE`

### 6.5 空池降级

如果当前 `pokemonPool[tier]` 为空：

- 持续降到更低 tier
- 直到 `COMMON`

### 6.6 传奇等级限制重抽

`Arena.checkLegendBST(species, adjustedWave)`

若种族是 `legendary / subLegendary / mythical`：

- `baseTotal >= 660` 时，`adjustedWave < 80` 需要重抽
- 否则 `adjustedWave < 55` 需要重抽

最多重抽 `10` 次。

`adjustedWave` 来自：

```text
gameMode.getWaveForDifficulty(waveIndex, true)
```

Daily 注释说明：

- `660+` 传奇理论上要到 Daily 50+ 才放行，但 Daily 最大波次目前只有 50
- `600~659` 档传奇要到 Daily 25+ 才放行

### 6.7 按等级替换野生形态

拿到原 species 后，会调用：

```text
species.getWildSpeciesForLevel(level, true, isBoss ?? isBossSpecies, gameMode)
```

如果返回的新 `speciesId` 与原 species 不同，则直接替换。

## 7. 遭遇阶段的敌方实例化

来源：

- `src/phases/encounter-phase.ts`
- `src/field/pokemon.ts`

### 7.1 训练家战

`EncounterPhase.start()`

- `battle.enemyParty[e] = battle.trainer?.genPartyMember(e)`

### 7.2 野生战

步骤：

1. 先按波次等级生成 species
2. 若玩家持有 `BoostBugSpawnModifier`
   - 当前不是 Boss 波
   - biome 不是 `END`
   - `randSeedInt(10) === 0`
   - 则以 `golden bug net` 池替换 species
3. 调用：

```text
addEnemyPokemon(species, level, TrainerSlot.NONE, 是否有 bossSegments)
```

### 7.3 最终 Boss 波修正

若当前 `battleSpec === FINAL_BOSS`：

- 敌方 IV 直接 `fill(31)`

### 7.4 Sync 同步性格

对玩家前排上场单位，从后往前遍历，执行：

```text
applyAbAttrs("SyncEncounterNatureAbAttr", { pokemon: playerPokemon, target: enemy })
```

## 8. Boss 血条段数

`BattleScene.getEncounterBossSegments(waveIndex, level, species?, forceBoss = false)`

### 8.1 override

- `ENEMY_HEALTH_SEGMENTS_OVERRIDE > 1`：直接返回 override
- `ENEMY_HEALTH_SEGMENTS_OVERRIDE === 1`：返回 `0`，表示不当 Boss 处理

### 8.2 Daily 最终 Boss

- dailyConfig 指定 `boss.segments` 时直接使用
- 否则默认 `5`

### 8.3 普通 Boss 判定

以下任一条件成立时 `isBoss = true`：

- `forceBoss`
- species 是 `subLegendary / legendary / mythical`
- `waveIndex % 10 === 0`
- 或随机 Boss 模式下：

```text
randSeedInt(100) < min(max(ceil((waveIndex - 250) / 50), 0) * 2, 30)
```

这条随机 Boss 概率从 250 波后开始增长，每 50 波增加 2%，上限 30%。

### 8.4 段数公式

Boss 成立后：

```text
ret = 2
if level >= 100: ret += 1
if species.baseTotal >= 670: ret += 1
ret += floor(waveIndex / 250)
```

### 8.5 双打 Boss 血条重分配

如果同一场双打里有多个 Boss，且没有 override：

```text
newSegments = ceil(originalSegments * (species.baseTotal / totalBst))
```

按双 Boss 总 BST 比例重新分配段数。

## 9. 敌方初始 IV 与好感

`EnemyPokemon.constructor()`

仅训练家拥有以下特殊初始化：

### 9.1 IV

训练家敌人每项 IV：

```text
randSeedIntRange(floor(waveIndex / 10), 31)
```

共 6 次。

### 9.2 好感

```text
friendship = clamp(round(255 * (waveIndex / 145)), 50, 255)
```

也就是：

- 最低 `50`
- 第 `145` 波达到 `255`

## 10. Endless / 特殊物种修正

`EncounterPhase.start()`

### 10.1 Eternatus

- 经典最终战或经典最终波：
  - 若不是 `FINAL_BOSS` 特规，则切到 `formIndex = 1`
  - 强制 `setBoss()`
- 每 `1000` 波：
  - `formIndex = 1`

### 10.2 训练家 / 野生遭遇结束后

普通遭遇在战斗前会：

- `regenerateModifierPoolThresholds(enemyField, TRAINER/WILD pool)`
- `generateEnemyModifiers()`
- `overrideModifiers(false)`
- `overrideHeldItems(enemy, false)`

即敌人的持有物和敌方修饰器是在遭遇阶段末尾统一生成。
