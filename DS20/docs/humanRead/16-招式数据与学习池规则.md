# PokeRogue 招式数据与学习池规则

本文档描述 `DS20/pokerogue-main` 中招式静态数据从哪里来、运行时招式对象如何组织、宝可梦可学招式池如何组成，以及敌方招式组是怎么生成的。

## 1. 招式数据的来源分层

招式相关数据分成四层：

1. 招式静态定义
   - 文件：`src/data/moves/move.ts`
   - 容器：`allMoves`
   - 初始化入口：`initializeGame() -> initMoves()`
2. 宝可梦单次携带的招式实例
   - 文件：`src/data/moves/pokemon-move.ts`
   - 类型：`PokemonMove`
3. 升级可学招式表
   - 文件：`src/data/balance/pokemon-level-moves.ts`
4. 蛋招 / TM 学习池
   - 蛋招：`src/data/balance/moves/egg-moves.ts`
   - TM 兼容表：`src/data/balance/tm-species-map.ts`
   - TM tier / 反向表：`src/data/balance/tms.ts`

## 2. 招式静态定义是怎么装载的

初始化路径：

```text
initializeGame()
-> initMoves()
-> allMoves.push(...)
```

`allMoves` 定义在：

- `src/data/data-lists.ts`

```ts
export const allMoves: readonly Move[] = [];
```

`initMoves()` 位于：

- `src/data/moves/move.ts`

它把所有 `new AttackMove(...)` / `new StatusMove(...)` / `new SelfStatusMove(...)` 等对象按 `MoveId` 顺序压进 `allMoves`。

## 3. 静态招式对象 `Move` 里有什么

`Move` 的核心字段包括：

- `id`
- `name`
- `type`
- `category`
- `moveTarget`
- `power`
- `accuracy`
- `pp`
- `chance`
- `priority`
- `generation`
- `attrs`
- `conditions`
- `conditionsSeq2`
- `conditionsSeq3`
- `restrictions`
- `flags`

也就是说，一条招式定义不仅包含基础面板值，也包含：

- 命中前失败条件
- 选择阶段限制
- 各种 move attrs
- 目标类型
- 标记位

## 4. 三种主要招式类型

### 4.1 `AttackMove`

构造参数：

```text
id, type, category, power, accuracy, pp, chance, priority, generation
```

内部默认：

- `moveTarget = NEAR_OTHER`

### 4.2 `StatusMove`

构造参数：

```text
id, type, accuracy, pp, chance, priority, generation
```

内部固定：

- `category = STATUS`
- `power = -1`
- `moveTarget = NEAR_OTHER`

### 4.3 `SelfStatusMove`

构造参数：

```text
id, type, accuracy, pp, chance, priority, generation
```

内部固定：

- `category = STATUS`
- `power = -1`
- `moveTarget = USER`

## 5. 招式实例 `PokemonMove`

`PokemonMove` 是宝可梦身上真正携带的招式实例。

它不复制整条静态招式定义，只保存：

- `moveId`
- `ppUsed`
- `ppUp`
- `maxPpOverride`

静态定义通过：

```text
getMove() => allMoves[this.moveId]
```

来取。

因此：

- `Move` 是全局模板
- `PokemonMove` 是每只宝可梦自己的带 PP 状态的实例

## 6. PP 的计算规则

`PokemonMove.getMovePp()`：

```text
maxPpOverride || basePp + ppUp * toDmgValue(basePp / 5)
```

也就是说：

1. 如果有 `maxPpOverride`，直接用它
2. 否则：
   - 从静态招式定义取 `pp`
   - 每层 `ppUp` 增加 `basePp / 5` 的伤害值式取整

`isOutOfPp()` 规则：

```text
getMovePp() !== -1 && ppUsed >= getMovePp()
```

## 7. 宝可梦当前招式组如何取得

`Pokemon.getMoveset(ignoreOverride = false)` 的顺序：

1. 先检查 `Overrides.MOVESET_OVERRIDE` / `Overrides.ENEMY_MOVESET_OVERRIDE`
2. 如果没有 override：
   - 若 `summonData.moveset` 存在且不忽略 override，则优先返回 `summonData.moveset`
   - 否则返回 `this.moveset`
3. 如果有 override：
   - 用 override 里的 `MoveId` 重建 `this.moveset`
   - 尽量保留原 `ppUsed`
   - 最后再按 `summonData.moveset` 优先级返回

## 8. 升级招式表来自哪里

升级招式表来源：

- `src/data/balance/pokemon-level-moves.ts`

这里维护两个主要表：

1. `pokemonSpeciesLevelMoves`
2. `pokemonFormLevelMoves`

表项结构：

```text
[学习等级, MoveId]
```

其中等级可以是：

- 正常等级 `1..N`
- `EVOLVE_MOVE`
- `RELEARN_MOVE`

`PokemonSpeciesForm.getLevelMoves()` 的规则：

1. 如果该物种当前 form 在 `pokemonFormLevelMoves` 中存在
   - 返回 form 专属表
2. 否则返回 `pokemonSpeciesLevelMoves[speciesId]`

## 9. 蛋招表来自哪里

蛋招表来源：

- `src/data/balance/moves/egg-moves.ts`

文件头已经写明：

- 不要手改
- 由 `eggMoves:parse` 脚本自动生成
- 输入源来自 Balance Team 的表格 CSV

运行时结构：

```ts
speciesEggMoves[rootSpeciesId] = [move0, move1, move2, move3]
```

这里固定是 4 个槽位：

- 前 3 个普通 egg move
- 第 4 个稀有 egg move

## 10. TM 数据来自哪里

TM 体系分两层。

### 10.1 正向兼容表

文件：

- `src/data/balance/tm-species-map.ts`

结构：

```text
tmSpecies[MoveId] = [SpeciesId 或 [SpeciesId, formKey1, formKey2...]]
```

也就是：

- key 是某个 TM 招式
- value 是哪些物种或哪些 form 能学

### 10.2 反向表与 tier

文件：

- `src/data/balance/tms.ts`

主要做两件事：

1. `transposeTmSpecies()` 把 `tmSpecies` 翻成：

```text
speciesTmMoves[speciesId] = [MoveId 或 [formKey, MoveId]]
```

2. `tmPoolTiers` 给每个 TM 招式标稀有度：
   - `COMMON`
   - `GREAT`
   - `ULTRA`

## 11. starter 自定义 moveset 的合法性

`PokemonSpecies.validateStarterMoveset(moveset, eggMoves)` 的规则：

对于玩家选进去的 starter moveset，每个招式必须满足以下条件之一：

1. 该招式在 `speciesEggMoves[rootSpeciesId]` 中存在，且对应 egg move 位已解锁
2. 该招式在当前 form 的升级表中存在，且学习等级 `<= 5`
3. 若没有 form 表，则该招式在物种基础升级表中存在，且学习等级 `<= 5`

所以 starter 初始招式只允许：

- 已解锁蛋招
- 5 级前可学招式

## 12. `getLevelMoves()` 怎么拼完整可学升级池

`Pokemon.getLevelMoves(...)` 的拼装规则：

1. 若 `learnSituation === EVOLUTION_FUSED` 且当前是 fusion：
   - 只取融合副体那一侧的升级招式
2. 否则：
   - 可选择是否 `simulateEvolutionChain`
   - 若开启，会沿整条模拟进化链收集招式
   - 若关闭，只取当前 form 的升级表
3. 如果当前是 fusion，且不是 `EVOLUTION_FUSED_BASE`：
   - 再把融合副体的升级招式也拼进去
4. 过滤条件：
   - 可选保留 `EVOLVE_MOVE`
   - 可选保留 `RELEARN_MOVE`
   - 不保留超过当前等级的招式
   - 不保留重复招式
5. 最终按学习等级排序输出

## 13. `getEggMoves()` 的规则

`Pokemon.getEggMoves()` 直接返回：

```text
speciesEggMoves[this.getSpeciesForm().getRootSpeciesId()]
```

也就是：

- 只看根物种
- 不看当前 form
- 不做运行时重算

## 14. 敌方招式组生成入口

敌方自动生成招式入口：

- `Pokemon.generateAndPopulateMoveset()`
- 真正算法在 `src/ai/ai-moveset-gen.ts`

普通情况：

```text
generateMoveset(this, useRivalSignatures)
```

特殊覆盖：

- Smeargle：4 个 `SKETCH`
- Eternatus：
  - form 0 一套固定招式
  - form 1 一套固定招式
  - 逆属性挑战时第 3 招额外替换

## 15. `generateMoveset()` 的完整流程

`generateMoveset(pokemon, forceRivalSignatures = false)` 的顺序：

1. `globalScene.movesetGenInProgress = true`
2. 清空 `pokemon.moveset`
3. 生成 level move pool
4. 如果是 trainer 或 boss：
   - 对 level pool 做 `filterSupercededMoves`
5. 如果是 trainer：
   - 生成 egg move pool
   - 对 egg pool 做 superseded 过滤
   - 生成 tm pool
   - 对 tm pool 做 superseded 过滤
6. 合并成 master pool：

```text
movePool = tmPool + eggMovePool + learnPool
```

7. 过滤禁用招式
8. trainer 额外调权
9. damaging move 再按威力和攻/特攻匹配度调权
10. 把权重做指数放大
11. 尝试强塞 signature move
12. 若没有合适 signature，则强塞一招 STAB
13. 剩余槽位按权重继续填充
14. `movesetGenInProgress = false`

## 16. 升级招式池权重规则

`getAndWeightLevelMoves()` 的权重规则：

1. 默认：

```text
weight = learnLevel + BASE_LEVEL_WEIGHT_OFFSET
```

`BASE_LEVEL_WEIGHT_OFFSET = 20`

2. 如果是 `EVOLVE_MOVE`：

```text
weight = EVOLUTION_MOVE_WEIGHT = 70
```

3. 如果是 1 级招式，且威力 `>= EVO_MOVE_BP_THRESHOLD`

```text
weight = RELEARN_MOVE_WEIGHT = 60
```

`EVO_MOVE_BP_THRESHOLD = 70`

4. 如果是 `RELEARN_MOVE`
   - trainer：`60`
   - 非 trainer：`0`

## 17. TM 可用等级门槛

定义在 `moveset-generation.ts`：

- `COMMON_TIER_TM_LEVEL_REQUIREMENT = 25`
- `GREAT_TIER_TM_LEVEL_REQUIREMENT = 40`
- `ULTRA_TIER_TM_LEVEL_REQUIREMENT = 55`

也就是：

- 25 级以下：不能带 TM
- 25-39：只能用 COMMON TM
- 40-54：COMMON + GREAT
- 55 级及以上：COMMON + GREAT + ULTRA

TM 权重：

- `COMMON_TM_MOVESET_WEIGHT = 12`
- `GREAT_TM_MOVESET_WEIGHT = 14`
- `ULTRA_TM_MOVESET_WEIGHT = 18`

## 18. Egg Move 可用等级门槛

定义在 `moveset-generation.ts`：

- `EGG_MOVE_LEVEL_REQUIREMENT = 80`
- `RARE_EGG_MOVE_LEVEL_REQUIREMENT = 170`

规则：

- 80 级以下：不能带 egg move
- 80-169：只能带前 3 个普通 egg move
- 170 级及以上：允许第 4 个 rare egg move
- boss 永远排除 rare egg move

egg move 权重：

```text
levelUpWeightedEggMoveWeight = round(max(levelPoolWeight) * EGG_MOVE_TO_LEVEL_WEIGHT)
eggMoveWeight = min(levelUpWeightedEggMoveWeight, EGG_MOVE_WEIGHT_MAX)
rareEggMoveWeight = eggMoveWeight * 5 / 6
```

常量：

- `EGG_MOVE_TO_LEVEL_WEIGHT = 0.85`
- `EGG_MOVE_WEIGHT_MAX = 60`

## 19. moveset 过滤规则

`filterMovePool()` 会删掉以下情况：

1. 权重 `<= 0`
2. 未实现招式
3. 有 `SacrificialAttrOnHit`
4. boss 的自爆 / Pain Split 类
5. trainer 的 OHKO 招式
6. trainer/boss 在单打时的双打限定招式
7. 达到阈值后的 `LEVEL_BASED_DENYLIST`
8. 已有天气/地形能力时的冲突设置招式
9. Gorilla Tactics 下的状态招式

## 20. trainer 额外调权规则

`adjustWeightsForTrainer()` 中：

1. 自爆类招式权重乘 `0.5`
2. 自身大幅强化招式权重乘 `1.25`
3. 蓄力招式 / Recharge 招式权重乘 `0.7`

## 21. damaging move 的调权规则

`adjustDamageMoveWeights()`：

1. 先计算所有伤害招式的有效威力
2. 取 `maxPower`，封顶 `MOVE_POWER_CEILING = 120`
3. 每个伤害招式先按：

```text
Clamp(power / maxPower, 0.25, 1)
```

缩放

4. 再看物攻 / 特攻匹配度
5. 如果招式用的是较差的输出端：
   - 额外乘以由攻/特攻比值得出的倍率
6. `Body Press` 类防御输出招式单独按防御比值调权

## 22. signature move 与 STAB 强插规则

### 22.1 signature move

`FORCED_SIGNATURE_MOVE_CHANCE = 60`

即有 `60%` 概率尝试塞入 signature move。

数据源：

- `src/data/balance/moves/signature-moves.ts`

规则：

1. 取该物种的 signature 配置
2. 若是单个 `MoveId`：
   - pool 里存在且 60% 命中才加入
3. 若是数组：
   - 先过滤出 pool 中存在的候选
   - 再 60% 命中
   - 从候选中随机一招

### 22.2 STAB move

如果没塞进合格 signature，或者塞进去的是状态招式 / 非伤害 STAB：

1. 尝试强塞一招 STAB 伤害招式
2. 部分招式在 `STAB_BLACKLIST` 里，不会用于这一步

## 23. 最终 moveset 的来源构成限制

最终 4 招来自以下来源的混合：

- level-up
- egg move
- TM
- signature 强插
- STAB 强插

同时还受：

- TM 数量上限
- egg move 数量上限
- superseded move 过滤
- forbidden move 过滤
- 当前双打/单打
- boss / trainer / wild 身份
- 当前等级

共同控制。
