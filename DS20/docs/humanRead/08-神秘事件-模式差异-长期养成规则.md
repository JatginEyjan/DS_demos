# 神秘事件、模式差异与长期养成规则

## 1. 神秘事件生成

来源：

- `src/battle-scene.ts`
- `src/data/mystery-encounters/*.ts`
- `src/phases/mystery-encounter-phases.ts`
- `src/system/game-data.ts`

## 1.1 合法波次

`BattleScene.isMysteryEncounterValidForWave(battleType, waveIndex)`

神秘事件生成要求同时满足：

- 模式开启了 `hasMysteryEncounters`
- 当前不是 Boss 波
- `waveIndex % 10 !== 1`
- 波次位于合法区间
- 当前 battleType 合法

对应模式区间来源：

- `CLASSIC_MODE_MYSTERY_ENCOUNTER_WAVES`
- `CHALLENGE_MODE_MYSTERY_ENCOUNTER_WAVES`

## 1.2 漏刷补偿

若一波本来可以刷神秘事件但没有刷出：

- `encounterSpawnChance += WEIGHT_INCREMENT_ON_SPAWN_MISS`

这一步发生在 session 保存之后。

## 1.3 遭遇创建

若 battle 标记为神秘事件战且 `mysteryEncounter` 还没装载：

- 用 `waveIndex * 16` 的 seed offset 加载事件对象

若事件定义了 `onInit()`：

- 在 `waveIndex` 的 seed offset 下执行

## 2. 神秘事件运行结构

神秘事件支持几种 encounter mode：

- 纯文本 / 纯选项
- `TRAINER_BATTLE`
- 战斗后奖励
- 战斗后商店
- 无奖励直接继续

事件对象可定义：

- `onInit`
- `introVisuals`
- `encounterAnimations`
- `onVisualsStart`
- `dialogue`
- `shop` / `rewards`
- 自定义战斗生成

## 3. Daily 模式差异

来源：

- `src/game-mode.ts`
- `src/data/daily-seed/daily-run.ts`

### 3.1 起始参数

- 起始等级 `20`
- 起始金钱优先读 daily seed
- 起始 biome 优先读 daily seed

### 3.2 难度波次

```text
difficultyWaveIndex = waveIndex + 30 + floor(waveIndex / 5)
```

### 3.3 训练家层

默认：

- 5, 15, 20, 25, 30, 35, 40, 45

但可被 `getDailyTrainerManipulation(waveIndex)` 改写。

### 3.4 最终 Boss

- 固定第 `50` 波
- species、form、nature、moveset、segments、是否可捕都可被 daily config 覆盖

### 3.5 Daily 起始 held item

`getDailyRunStarterModifiers(party)`

每只起始宝可梦会抽 `3` 个 held item。

稀有度抽法：

```text
tierValue = randSeedInt(64)
>25 => COMMON
>12 => GREAT
>4  => ULTRA
>0  => ROGUE
=0  => MASTER
```

Daily starter 池：

- `COMMON`: `BASE_STAT_BOOSTER`, `BERRY`
- `GREAT`: `ATTACK_TYPE_BOOSTER`
- `ULTRA`: `REVIVER_SEED`, `SOOTHE_BELL`, `SOUL_DEW`, `GOLDEN_PUNCH`
- `ROGUE`: `GRIP_CLAW`, `BATON`, `FOCUS_BAND`, `QUICK_CLAW`, `KINGS_ROCK`
- `MASTER`: `LEFTOVERS`, `SHELL_BELL`

## 4. 挑战模式挂钩点

系统不是把挑战放在单一入口处理，而是散布在各模块调用 `applyChallenges(...)`。

当前核心挂钩点包括：

- `FIXED_BATTLES`
- `SHOP`
- `SHOP_ITEM`
- `WAVE_REWARD`
- `PARTY_HEAL`
- `PASSIVE_ACCESS`
- `FLIP_STAT`
- `FRESH_START`
- `TERA_ACCESS`
- `PASSIVES`

实际效果包括：

- 改固定战
- 改奖励合法性
- 禁商店
- 禁队伍恢复
- 反转能力值
- 关闭 passive
- 限制捕捉与图鉴成长

## 5. 初始队伍与长期数据

来源：

- `src/system/game-data.ts`
- `src/system/pokemon-data.ts`
- `src/system/voucher.ts`
- `src/data/balance/starters.ts`

### 5.1 starter 长期数据

`starterData[speciesId]` 至少长期记录：

- `friendship`
- `candyCount`
- 解锁情况
- 闪光 / 变种可用性
- 蛋招式记录

### 5.2 starter 成本与好感阈值

`getStarterValueFriendshipCap(starterCost)`

- `1` 费：25
- `2` 费：50
- `3` 费：75
- `4` 费：100
- `5` 费：150
- `6` 费：200
- `7` 费：300
- `8/9` 费：450
- 其他：600

到达阈值就按整除数量发 starter candy。

## 6. 存档分层

`GameData` 把数据拆成系统数据和 session 数据。

### 6.1 session 里保存的核心内容

- 当前 battle
- 波次号
- 当前 biome
- 玩家队伍
- 敌方队伍
- money
- pokeballCounts
- modifiers
- enemyModifiers
- arena 状态

### 6.2 存档恢复

从 session 恢复 battle 时：

- 直接读取 waveIndex / battleType / trainer / mysteryEncounterType
- 敌我单位按 dataSource 恢复
- 然后重新进入当前 battle 的 phase 流

## 7. voucher

`battle-scene.validateVoucher(voucher, args?)`

触发条件满足且此前未解锁时：

1. `voucherUnlocks[voucher.id] = Date.now()`
2. UI 显示 voucher
3. `voucherCounts[voucher.voucherType]++`

voucher 不属于单局即时资源，而是长期账号资源。

## 8. 蛋与孵化

来源：

- `src/data/egg.ts`
- `src/phases/egg-hatch-phase.ts`
- `src/data/balance/rates.ts`

### 8.1 gacha 蛋稀有度阈值

随机值范围 `0~255`，比较阈值：

- `COMMON` 阈值：52
- `RARE` 阈值：8
- `EPIC` 阈值：1
- `LEGENDARY`：默认剩余的 `1/256`

Legendary Up 会把这些阈值整体上移 `1`，使 Legendary 蛋概率变为 `2/256`。

### 8.2 保底

- Legendary：412 抽
- Epic：59 抽
- Rare：9 抽

### 8.3 孵化所需波次

- Common：10
- Rare：25
- Epic：50
- Legendary：100
- Manaphy：50

### 8.4 闪率

- 默认 gacha：`1/128`
- shiny up gacha：`1/64`
- 同 species 蛋：`1/12`

### 8.5 隐藏特性率

- 同 species 蛋：`1/8`
- gacha 蛋：`1/192`

## 9. 图鉴、捕获与长期解锁

一次成功捕捉或孵化会更新：

- `dexData` 的 seen / caught
- `starterData`
- 相关成就
- 相关 voucher 校验
- 对应物种线的 ribbon / candy / friendship 累积

这些数据都属于长期进度，不随单局结束清空。
