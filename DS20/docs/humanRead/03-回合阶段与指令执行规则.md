# 回合阶段与指令执行规则

## 1. 阶段容器

来源：

- `src/phase-manager.ts`
- `src/phases/*.ts`
- `src/queues/*.ts`

战斗不是一个函数循环，而是由 `PhaseManager` 串联多个 `Phase`。

一次标准波次的主路径是：

1. `NewBattlePhase`
2. `EncounterPhase / NewBiomeEncounterPhase / NextEncounterPhase`
3. `InitEncounterPhase`
4. `CommandPhase / EnemyCommandPhase`
5. `TurnStartPhase`
6. `MovePhase / Switch / Capture / Run`
7. 各类后置 phase
8. `VictoryPhase` 或继续下一回合

## 2. TurnInitPhase

`TurnInitPhase.start()`

执行顺序：

1. 检查玩家上场单位是否合法参与当前挑战
2. 非法进场单位处理：
   - 若后排有合法替补：强制 `switchOut()`
   - 若没有合法替补但还未 Game Over：`leaveField()`
   - 若全队都不合法：清空 phase queue，进入 `GameOverPhase`
3. 触发 `TurnInitEvent`
4. 执行神秘事件战斗开局效果
5. 若神秘事件要求跳过本轮，其余 phase 不再入队
6. 遍历场上单位
   - 玩家单位：加入 `playerParticipantIds`
   - 所有 active 单位：`resetTurnData()`
   - 玩家入 `CommandPhase`
   - 敌方入 `EnemyCommandPhase`
7. 最后压入 `TurnStartPhase`

## 3. 指令录入

### 3.1 玩家指令

`CommandPhase`

可录入的主指令：

- `FIGHT`
- `BALL`
- `POKEMON`
- `RUN`
- `TERA`（在 `preTurnCommands` 中处理）

### 3.2 BALL 限制

`CommandPhase.checkCanUseBall()`

以下情况不能丢球：

- 训练家战
- `END` biome 且不满足允许捕捉的特殊规则
- 神秘事件明确禁止捕捉

`END` biome 里的关键限制：

- 经典模式非最终 Boss 波且场上有未捕获物种：禁用
- Full Fresh Start 且不是经典最终 Boss 波：禁用
- Endless 且不是 Endless minor boss：禁用
- Daily 最终 Boss 若配置 `catchable = false`：禁用

### 3.3 RUN 限制

`CommandPhase.handleRunCommand()`

以下情况不能逃跑：

- `END` biome
- 训练家战
- 神秘事件训练家战
- 神秘事件设置 `fleeAllowed = false`
- 宝可梦被招式/特性/标签阻止离场

## 4. TurnStartPhase 排序

### 4.1 指令大优先级

`TurnStartPhase.getCommandOrder()`

排序规则第一层：

- 非 `FIGHT` 指令优先于 `FIGHT`
- 即 `SWITCH / BALL / RUN` 一定先于攻击动作入队

### 4.2 Tera 前置

在正式处理 turn command 前，先按速度遍历场上单位：

- 读取 `preTurnCommands`
- 如果 command 为 `TERA`，先推入 `TeraPhase`

### 4.3 Bypass Speed 检查

对所有本回合执行 `FIGHT` 的单位，按速度顺序依次应用：

- `BypassSpeedChanceAbAttr`
- `BypassSpeedChanceModifier`

## 5. 速度队列

来源：

- `src/utils/speed-order.ts`
- `src/utils/speed-order-generator.ts`
- `src/queues/pokemon-priority-queue.ts`
- `src/queues/move-phase-priority-queue.ts`

### 5.1 速度排序过程

`sortInSpeedOrder()`

步骤：

1. 先把连续由同一宝可梦产生的动作分组成一个 group，避免自我速度平局打乱先后
2. 对 group 做随机洗牌
   - 种子偏移：`currentBattle.turn * 1000 + pokemonList.length`
   - 波次 seed：`waveSeed`
3. 按速度排序
   - 速度值取 `getEffectiveStat(Stat.SPD)`
   - 默认从高到低
4. 若场上有 `TRICK_ROOM`
   - 整体反转顺序

### 5.2 固定回合顺序

如果 `turnCommandManager.setOrder` 已设置：

- 完全按该数组顺序排序
- 不再按速度计算

## 6. MovePhase 二次排序

`MovePhasePriorityQueue.sortPostSpeed()`

在速度序基础上，再做三层排序：

1. `timingModifier`
2. 招式 priority
3. 招式括号内 priority modifier

比较器：

```text
timingModifier 降序
priority 降序
priorityModifier 降序
```

## 7. 指令转 phase

`TurnStartPhase.handleTurnCommand()`

### 7.1 FIGHT

- 校验招式是否还能从 moveset 中找到并且还有 PP
- 找不到时 fallback 为 `new PokemonMove(queuedMove.move)`
- 有 `MoveHeaderAttr` 时先插入 `MoveHeaderPhase`
- 再推入 `MovePhase(pokemon, targets, move, useMode)`

### 7.2 BALL

- 直接 `unshiftNew("AttemptCapturePhase", targetIndex, pokeballType)`
- 因为 `unshift`，所以优先级高于后续普通 phase

### 7.3 POKEMON

- `unshiftNew("SwitchSummonPhase", switchType, fieldIndex, cursor, true, isPlayer)`

### 7.4 RUN

- `unshiftNew("AttemptRunPhase")`

## 8. 逃跑成功率公式

`AttemptRunPhase.calculateEscapeChance(escapeAttempts)`

基础数据：

- `enemySpeed = 敌方场上单位基础速度之和`
- `playerSpeed = 玩家场上单位基础速度之和`
- 注意这里取的是 `getStat(Stat.SPD)`，不是有效速度

### 8.1 Boss / 非 Boss 两套参数

若敌方场上至少 1 个 Boss：

- `speedCap = 6`
- `minChance = 5`
- `maxChance = 25`
- `escapeBonus = 2`

否则：

- `speedCap = 4`
- `minChance = 5`
- `maxChance = 95`
- `escapeBonus = 10`

### 8.2 速度比例

```text
speedRatio = playerSpeed / enemySpeed
escapeSlope = (maxChance - minChance) / speedCap
chance = minChance + min(speedRatio, speedCap) * escapeSlope + escapeAttempts * escapeBonus
finalChance = clamp(chance, minChance, maxChance)
```

然后对玩家侧按速度遍历，应用 `RunSuccessAbAttr` 修改最终值。

实际判定：

```text
escapeRoll = randBattleSeedInt(100)
escapeRoll < escapeChance
```

### 8.3 失败后果

逃跑失败时：

- 玩家前排 `turnData.failedRunAway = true`
- 回合继续

## 9. 捕捉流程阶段

`AttemptCapturePhase.start()`

执行顺序：

1. 扣除球数量
2. 计算修正捕获率
3. 计算摇动概率与 critical capture 概率
4. 投球动画
5. 入球动画
6. 摇球校验
7. 成功则 `catch()`
8. 失败则 `failCatch()`

详细公式见 `05-成长-经验-等级上限-进化-捕捉规则.md`

## 10. 胜负与回合尾部

`TurnStartPhase.start()` 尾部固定入队：

1. `CheckInterludePhase`
2. `queueTurnEndPhases()`

说明：

- 各种 `unshift` 产生的 phase，例如 `SwitchSummonPhase`、`AttemptRunPhase`、`AttemptCapturePhase`
  会插到队列前方，优先执行
- 因此指令录入顺序与 phase 真正执行顺序不是一一对应的

## 11. Boss 受伤与分段

`EnemyPokemon.damage()`

Boss 在 `ignoreSegments = false` 时，先按血条段数切分伤害：

```text
segmentSize = maxHp / bossSegments
```

调用 `calculateBossSegmentDamage(...)`，保证一次伤害不会跨越不允许穿透的段位。

特殊规则：

- 经典最终 Boss 第一形态在 `bossSegmentIndex < 1` 时，伤害最多打到剩 `1` HP，不允许直接收掉

### 11.1 破段增益

`EnemyPokemon.handleBossSegmentCleared(segmentIndex)`

仅野生 Boss 触发 stat boost，训练家 Boss 默认不触发。

每破一段：

1. 从未满级的有效能力中选一个
2. 权重等于该能力当前永久数值
3. 默认提升 `+1`
4. 若总段数 `>= 3` 且进入最后一段，额外 `+1`
5. 若总段数 `>= 5` 且进入倒数第二段，额外 `+1`

即：

- 3 段 Boss 最后一段会给 `+2`
- 5 段 Boss 倒数第二段和最后一段都会给 `+2`
