# DS11 迷雾分支网格方案确认文档

**版本**: v0.2  
**日期**: 2026-02-21  
**目标**: 方案确认 + 设计疑虑梳理 + 原型开发准备

---

## 一、核心方案确认

### 1.1 网格形式：迷雾生长 + 分支抉择

**基础结构**
```
初始：中心格已揭示，周围8格为"迷雾边缘"
生长规则：揭示迷雾边缘格 → 该格显示信息 → 其相邻迷雾成为新边缘
特殊格"深渊裂隙"：揭示后向3个方向同时打开新路径
分支枯萎：未选择的其他方向在N回合后消失
```

**关键特性**
- 网格不是固定的，是动态生成的
- 迷雾有流动性（边缘可能移位）
- 分支选择不可逆（枯萎后永远失去）
- 像树一样从中心向外生长，但生长方向不确定

### 1.2 揭示形式：代价派遣（暂用基础版）

**简化版机制**
```
- 消耗行动点派遣单位揭示格子
- 派遣有成功率（不是100%）
- 派遣需要时间（等待结果）
- 失败可能踩雷触发F3
```

**暂不实现（后续迭代）**
- 派遣者背叛机制
- 渐进信息层级（触摸感应）
- 派遣者永久死亡

## 二、关键设计疑虑与解决方案

### 疑虑1：迷雾流动让玩家感到"被耍"

**问题描述**
```
玩家标记的安全格，突然因为迷雾流动变成未知
玩家会觉得"我之前的努力白费了"
```

**解决方案**
```
A. 流动可预测化
   - 固定每5回合流动一次
   - 流动前1回合给出预警（迷雾开始"蠕动"）
   - 玩家可以计算"安全窗口期"

B. 流动有限制
   - 只流动"边缘迷雾"，已揭示格不受影响
   - 或：已揭示格有"稳定期"（揭示后3回合内不会流动）
   - 玩家可以利用这个窗口期

C. 流动成为工具
   - 玩家可以主动触发流动（消耗资源）
   - 用于切断追兵、刷新不利地形
   - 从"被动受害"变成"主动利用"
```

**推荐方案：A + B**
- 流动可预测 + 已揭示格受保护
- 保留紧张感，但玩家有掌控感

---

### 疑虑2：分支枯萎造成"被迫rush"的压迫感

**问题描述**
```
玩家正在思考走哪条路，倒计时在走
感到被游戏推着走，没有思考空间
```

**解决方案**
```
A. 延长枯萎时间
   - 从3-5回合延长到8-10回合
   - 给玩家充足思考时间

B. 延缓机制
   - 玩家可以消耗资源"标记"一个分支延缓枯萎
   - 延缓不是阻止，是争取时间
   - 成本递增（第一次1资源，第二次2资源...）

C. 分支复活（可选）
   - 某些道具/能力可以让已枯萎的分支重新生长
   - 但成本高昂
   - "不是永远失去，只是暂时错过"
```

**推荐方案：A + B**
- 足够长的默认时间 + 可付费延缓
- 紧张但不压迫

---

### 疑虑3：生长随机性导致"运气决定一切"

**问题描述**
```
随机生长可能导致：
- 某局全是死路
- 某局直接开到捷径
玩家觉得"不是技术，是运气"
```

**解决方案**
```
A. 生长有保底
   - 每生长3个普通格，必定出现1个"深渊裂隙"（分支点）
   - 确保玩家有选择空间

B. 难度梯度
   - 浅层：雷率低，资源丰富
   - 深层：雷率高，资源极好
   - 玩家自己决定深入多少

C. 预览机制
   - 派遣单位"感知"时，可以看到迷雾下的"大致轮廓"
   - 不是完全盲，是有信息的赌博
```

**推荐方案：A + B + C**
- 保底机制 + 难度梯度 + 预览信息
- 随机但有边界，技术+运气结合

---

### 疑虑4：单局时长难以控制

**问题描述**
```
生长机制可能导致：
- 某局很快找到出口（2分钟）
- 某局一直生长没尽头（20分钟）
目标：3-5分钟一局难以保证
```

**解决方案**
```
A. 强制出口机制
   - 每局必定在生长到第15个格时出现"出口"选项
   - 玩家可以选择离开，或继续深入
   - 保证有退出点

B. 深度限制
   - 最多生长3层（浅/中/深）
   - 到达深层后不再生长，必须做选择

C. 资源限制
   - 行动点有限（比如10点）
   - 用完强制结束（被困，损失部分资源）
   - 自然限制时长
```

**推荐方案：A + C**
- 强制出口选项 + 行动点限制
- 玩家掌控节奏，但有上限

---

### 疑虑5：没有派遣背叛，是否缺少深度

**问题描述**
```
简化版派遣只有成功/失败
缺少"带回错误信息"的戏剧张力
```

**解决方案（MVP后迭代）**
```
MVP版本先实现基础派遣：
- 成功：带回正确信息
- 失败：踩雷触发F3，派遣者受伤

V1.1版本加入：
- 高印记派遣者可能带回模糊信息（数字±1误差）
- 玩家需要交叉验证

V1.2版本加入：
- 畸变后的派遣者可能完全错误
- 但玩家可以通过派遣者状态预判
```

**推荐：分阶段实现**
- MVP先验证核心loop
- 后续迭代增加信息层级深度

## 三、MVP原型方案

### 3.1 核心机制（必须实现）

```
✅ 迷雾生长网格
   - 3×3初始，最大生长到15格
   - 每3个普通格后必出1个分支点
   - 分支枯萎时间：8回合

✅ 基础派遣揭示
   - 消耗1行动点派遣
   - 成功率80%（老兵角色）
   - 派遣耗时2回合
   - 失败触发F3

✅ F3强制揭示
   - 踩雷后揭示周围3格
   - 连锁反应（如果揭示的格也是雷）

✅ F4印记系统
   - 踩雷获得1印记
   - 3印记触发畸变选择（3选1）
   - 畸变持续本局

✅ 行动点系统
   - 初始10点
   - 揭示/派遣/移动都消耗
   - 用完=被困（游戏结束）

✅ 胜利条件
   - 找到"深渊核心"（藏在某格下方）
   - 找到出口（固定第15格出现）
   - 成功撤退
```

### 3.2 暂不实现（后续迭代）

```
❌ 迷雾流动（机制复杂，MVP后加）
❌ 派遣背叛（需要更多信息层级）
❌ 多角色切换（先单角色验证）
❌ 局外Meta（先验证单局好玩）
❌ 复杂畸变（先做3种简单畸变）
```

### 3.3 开发计划（5天）

```
Day 1: 迷雾生长网格
   - 基础3×3 → 生长算法 → 分支点生成

Day 2: 派遣揭示系统
   - 派遣按钮 → 等待机制 → 成功率判定

Day 3: F3+F4整合
   - 踩雷触发 → 连锁揭示 → 印记累积 → 畸变选择

Day 4: 行动点与胜利条件
   - 消耗系统 → 核心隐藏 → 出口生成 → 撤退结算

Day 5: 调优与测试
   - 数值平衡 → 体验调优 → Bug修复
```

### 3.4 验证标准

```
核心问题：玩家是否会主动踩雷触发F3+F4？

验证方法：
- 观察测试玩家行为
- 统计主动踩雷 vs 避免踩雷的比例
- 如果>30%的玩家主动踩雷 → 核心loop成立
- 如果<10% → 需要重新设计F3+F4的奖励机制

辅助指标：
- 单局时长是否集中在3-5分钟
- 玩家是否愿意玩3局以上
- 玩家是否讨论"刚才我应该选另一条路"
```
