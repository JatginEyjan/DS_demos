# DS11 设计文档 v0.1
## 起于扫雷的Roguelike原型

**文档版本**: 0.1  
**创建时间**: 2026-02-21  
**文档定位**: 核心设计框架，后续设计的参考基准

---

## 一、设计原点

### 1.1 核心命题
> **起于扫雷，不止于扫雷**

不是对扫雷的改良，而是以扫雷为起点的全新玩法探索。

### 1.2 核心体验目标
**"在信息不完全的空间中，管理风险并贪婪地收集资源"**

与传统扫雷的差异：
- 扫雷：解谜 → 完美解法存在
- DS11：博弈 → 没有完美解法，只有足够好的选择

## 二、核心机制框架（F3+F4生态系统）

### 2.1 机制定位

**F3（强制揭示）：风险加速器**
- **本质**：踩雷不是终点，是触发连锁事件的起点
- **作用**：将原本可通过探索规避的风险，强制摆到玩家面前
- **体验**：玩家被迫面对未知，失去"完全掌控"的幻觉

**F4（印记-畸变）：能力转化器**
- **本质**：印记不是惩罚计数，是角色的第二资源池
- **作用**：累积印记触发畸变，改变角色的玩法风格
- **体验**：从"避免受伤"到"管理受伤的方式"

### 2.2 系统耦合关系

```
踩雷（触发）
    ↓
F3强制揭示（创造新情境）
    ↓
玩家被迫应对（决策点）
    ↓
获得印记（F4累积）
    ↓
印记满触发畸变（能力转化）
    ↓
新的玩法风格（长期策略）
```

**关键洞察**：
> F3创造"危机情境"，F4提供"危机应对工具"，两者结合让玩家在被逼入困境后，用畸变能力寻找出路。

### 2.3 设计原则

**原则1：风险即机遇**
- 踩雷不是失败，是触发新情境的入口
- 强制揭示可能暴露捷径、资源或出口

**原则2：印记即资源**
- 印记不是倒计时，是可利用的第二能力池
- 不同角色有不同的印记经济学（获取/消耗/转化）

**原则3：畸变即转化**
- 畸变不是弱化，是玩法风格的改变
- 畸变能力可能是负面的、正向的或彩蛋性质的

**原则4：错误成本可控**
- 从"即死"到"渐进惩罚"
- 玩家有"回应权"——可以选择如何应对连锁事件

## 三、空间设计（网格形式）

### 3.1 网格的本质
**"选择空间的拓扑结构"**

突破传统扫雷的"均匀2D平面"假设，探索不均匀的空间形式。

### 3.2 网格形式选项

#### 形式A：多层立体网格
```
Layer 1（表层）：5×5，信息清晰，资源少
Layer 2（中层）：4×4，信息模糊，资源中等  
Layer 3（深层）：3×3，信息混乱，资源极好

连接：特定格子是"电梯"（通往下一层）
F3效果：踩雷可能"震塌"上层或"震出"下层入口
```
**核心体验**：垂直深入的勇气考验，越深信息越不可靠但诱惑越大

#### 形式B：非欧几里得网格
```
表面是5×5，但实际连接不规则
- A1→A2可能出现在B3位置
- 某些格子是"传送门"
- 踩雷后格子位置重新洗牌
```
**核心体验**：空间本身在欺骗，地图不可靠

#### 形式C：动态生长网格
```
初始只有3×3中心区域可见
边缘是"迷雾"
每揭示边缘格，迷雾退散，露出新格子
网格形状不规则生长
```
**核心体验**：不是在探索固定地图，是在唤醒沉睡的空间

#### 形式D：多分支树状网格
```
起始点是根
每个揭示的格子有2-3个方向选择
形成树状结构
但分支会"枯萎"（时间/资源限制，不能探索所有分支）
```
**核心体验**：必须选择走哪条路，永远不知道未选择的分支里有什么

### 3.3 网格与F3的耦合

**关键设计**：F3的强制揭示效果应该与网格拓扑耦合

| 网格形式 | F3连锁形态 |
|---------|-----------|
| 多层立体 | 震塌/震出跨层连接 |
| 非欧几里得 | 空间扭曲，格子位置重排 |
| 动态生长 | 加速/减速迷雾退散 |
| 多分支树状 | 强制打开封闭分支或枯萎开放分支 |

## 四、信息获取设计（揭示形式）

### 4.1 揭示的本质
**"信息获取的交互仪式"**

突破传统"点击=立即完整信息"的假设，引入渐进、代理、代价等维度。

### 4.2 揭示形式选项

#### 形式A：触摸感应（渐进）
```
手指按住格子
→ 显示"触感"（安全=温暖，危险=刺痛）
→ 继续按住，显示"声音"（安全=安静，危险=低语）
→ 最终松开，显示数字
```
**核心体验**：用感官探索，手指触觉比视觉更早感知危险

#### 形式B：派遣调查（代理）
```
派遣虚拟单位去调查
- 单位移动需要时间
- 单位可能被"吞噬"（如果格是雷）
- 单位带回的信息可能有误（如果单位受损）
```
**核心体验**：指挥而非亲自涉险，探子没回来=那个格有问题

#### 形式C：共鸣揭示（连锁）
```
揭示一个格
→ 与已揭示格"共鸣"
→ 如果数字有逻辑关系，连线高亮
→ 玩家通过"连接线索"推理隐藏格
```
**核心体验**：每个信息都是拼图的一部分，发现关联时整个区域清晰

#### 形式D：代价揭示（交易）
```
揭示需要支付代价：
- 支付行动点（标准）
- 支付生命值（冒险）
- 支付信息（用已知的格换未知的格）
```
**核心体验**：信息是有价格的，要权衡值不值得

### 4.3 揭示与F4的耦合

**关键设计**：印记状态影响揭示的可靠性

- 低印记：揭示信息准确
- 高印记：揭示信息可能出现误差（数字跳动、触感模糊）
- 畸变后：某些畸变能力改变揭示方式（如"深渊感知"可感知1格外的雷）

## 五、核心Loop设计

### 5.1 单局目标
**在调查员完全畸变前，收集足够"深渊核心"并找到出口撤退**

### 5.2 标准流程

```
1. 选择网格形式（多层/非欧/动态/树状）
2. 选择揭示形式（触摸/派遣/共鸣/代价）
3. 进入随机生成网格
4. 消耗"行动点"进行揭示或移动
5. 安全格：获取资源/信息
6. 踩雷格：
   - 触发F3强制揭示（连锁反应）
   - 获得印记
7. 印记满3个：触发畸变选择
8. 收集足够核心：寻找出口
9. 撤退：带回资源用于局外解锁
```

### 5.3 关键决策时刻

**时刻1：初入的贪婪**
- 看到深层诱惑，但信息模糊
- 决策：稳扎稳打 vs 冒险深入

**时刻2：踩雷的应对**
- F3强制揭示创造新情境
- 决策：冲向暴露的捷径 vs 处理新危险 vs 撤退

**时刻3：印记的权衡**
- 获得印记，能力开始变化
- 决策：避免再踩雷 vs 主动追求印记触发畸变

**时刻4：畸变的选择**
- 3个印记满，必须选择畸变方向
- 决策：根据当前处境选择最适合的畸变能力

**时刻5：撤退的赌博**
- 核心在手，但回去的路充满未知
- 决策：硬冲 vs 谨慎绕路 vs 使用畸变能力

### 5.4 体验张力

| 张力 | 描述 |
|------|------|
| 信息 vs 行动 | 知道越多，行动点消耗越多；不知道就盲目前进可能踩雷 |
| 安全 vs 贪婪 | 在1层稳扎稳打安全但收益低，去3层收益高但风险大 |
| 印记 vs 畸变 | 踩雷获得印记（负面），但畸变可能获得能力（正面） |
| 当下 vs 未来 | 这局成功撤退，但调查员已畸变，下次带着印记进来 |

## 六、畸变系统设计

### 6.1 畸变类型

**类型A：负面畸变（容错向）**
- 减少印记上限但获得稳定性
- 例："麻木"——印记上限降为2，但揭示信息永远不会错误

**类型B：正向畸变（能力向）**
- 获得特殊能力但承担风险
- 例："深渊之眼"——可以看到所有雷的位置（5秒），但每次使用失去1行动点

**类型C：彩蛋畸变（剧情向）**
- 解锁隐藏内容或特殊叙事
- 例："低语理解者"——可以听懂网格中的背景低语，获得额外线索

### 6.2 角色与畸变的耦合

不同角色有不同的畸变倾向：

| 角色类型 | 畸变特点 |
|---------|---------|
| 容器型 | 可以承载更多印记才畸变，但畸变后负面效果更强 |
| 转化型 | 印记满时可选择"主动畸变"获得强化能力 |
| 共鸣型 | 印记影响队友（团队共享印记池），可分工承担 |
| 利用型 | 某些能力需要消耗印记才能使用，印记=资源 |

## 七、后续设计方向

### 7.1 待细化系统

1. **网格生成算法**：如何保证每局有趣？难度曲线如何设计？
2. **行动点-资源经济**：如何获得？如何消耗？平衡性？
3. **数值框架**：揭示成本、印记累积速度、畸变触发条件
4. **局外Meta**：带回的资源如何用于解锁？长期成长曲线？

### 7.2 验证优先级

**第一验证目标**：
> 玩家是否会在安全时，主动选择"踩雷"来触发F3+F4？

如果会 → 核心Loop成立，F3+F4是核心玩法
如果不会 → F3+F4只是惩罚系统，需要重新设计

**MVP验证方案**：
- 多层立体网格（3层）
- 触摸感应揭示
- 1个角色（转化型）
- 3种畸变选项
- 单局目标：收集1个核心并撤退

## 八、设计边界与约束

### 8.1 我们要做的
- 保持单局轻量（3-5分钟）
- 每局随机生成，确保重玩性
- F3+F4作为核心生态系统，不是附加机制
- 让玩家感到"我想再来一局"而非"我终于过了"

### 8.2 我们不做的
- 复杂的剧情叙事（不是内容驱动）
- 多人实时对战（保持单机体验）
- 永久死亡导致角色完全丢失（保留部分进度）
- 精确计算的最优解（保留不确定性）
