# v0.1.2 界面修改计划

> **项目代号**：DS18
> **基线版本**：v0.1.1
> **目标版本**：v0.1.2
> **最后更新**：2026-04-02
> **目标**：不扩玩法系统，集中完善界面表现、信息结构与交互节奏，使当前原型更贴近 GDD 中的“信息与 UI 设计”。

---

## 一、版本定位

`v0.1.2` 定位为一个 **UI 体验校准版**。

本版本不继续增加新玩法深度，而是围绕以下目标展开：

- 让界面从“功能原型”转向“可感知为游戏的原型”
- 让规划阶段、执行阶段、结算阶段具有清晰不同的界面气质
- 让信息层级贴近 GDD 中“确定 / 模糊 / 隐藏 / 迷雾”的设计
- 让玩家不依赖长文本，也能快速理解当前风险、收益与推进状态

---

## 二、与 GDD 的核心差距

对照 [03-GDD游戏设计文档.md](./03-GDD游戏设计文档.md) 第 12 节，当前版本的主要偏差如下：

### 1. 执行阶段不够像“游戏中的战斗过程”

GDD 要求执行阶段具备：

- 顶部状态栏
- 中央当前卡槽场景
- 底部当天时间轴推进条
- 可展开的日程规划表

当前实现更接近：

- 一张当前卡描述卡
- 一列执行状态列表
- 右侧消耗品面板

问题在于：

- 缺少“中央舞台”
- 缺少“勇者正在推进”的视觉反馈
- 缺少场景、敌人、掉落、NPC 的现场感

### 2. 顶部状态信息分散，不像战斗 HUD

GDD 中顶部状态栏的重点是：

- HP 条
- 灵魂数
- 治疗瓶数
- Boss 逼近度

当前这些信息分散在多个 dashboard 卡片中，虽然功能完整，但不够像游戏中的高优先级 HUD。

### 3. 规划阶段仍偏“管理面板”，而不是“命运编排台”

GDD 中的规划阶段强调：

- 左侧背包
- 中央视觉化时间轴编辑区
- 右侧勇者状态面板
- 底部日程规划表

当前版本虽已是三栏结构，但中央时间轴的“编排感”和底部日程表的“路线感”仍然不强。

### 4. 信息可见性层级尚未落地

GDD 明确要求：

- 确定信息：详细展示
- 模糊信息：用星级 / 条形图展示
- 隐藏信息：执行时揭示
- 迷雾信息：夜晚房间只显示大致危险度和未知内容

当前版本大多数信息仍以文字直接平铺，缺少层级区分。

### 5. 界面视觉风格统一度不足

当前样式已具备暗黑奇幻基调，但整体仍偏“高保真后台”：

- 卡片结构均质
- 层级节奏接近管理系统
- 场景氛围和玩法语义之间绑定不够强

## 三、v0.1.2 总体目标

### 目标 A：执行阶段游戏化

让执行阶段从“阅读状态”变为“观看推进”。

核心变化：

- 用中央 Encounter Stage 替代单纯说明卡
- 强化当前房间、敌人、NPC、事件、掉落的舞台感
- 用底部推进时间轴替代列表式“已完成/待执行”
- 加入速度控制与轻量动态反馈

### 目标 B：规划阶段编排化

让规划阶段更像“构筑路径与命运”的操作台。

核心变化：

- 背包按三大类 tab 切换
- 中央时间轴强化昼夜切分、时长占用、风险提示
- 右侧显示更强的房间预测与角色构筑信息
- 底部加入多日规划条，展示锁定 / 当前 / 未来预布局

### 目标 C：信息层级可视化

让玩家看到的不是一堆说明文字，而是“可快速扫读的游戏信息”。

核心变化：

- 风险与收益改用图形化表达
- 夜晚信息迷雾化
- 隐藏内容执行时揭示
- 关键资源始终在高优先级区域可见

### 目标 D：视觉表现更贴近 GDD 气质

让 DS18 呈现出更明确的：

- 暗黑桌游感
- 魂味战役感
- “编排命运”而非“整理卡牌”的主题感

## 四、版本拆分建议

建议将 `v0.1.2` 按 3 个小包推进。

### v0.1.2-a 信息结构重组

目标：先把页面结构调整对。

范围：

- 重做全局壳层布局
- 统一顶部 HUD
- 加入底部多日规划条 / 当天推进条
- 调整日志面板定位与权重

验收标准：

- 执行、规划、结算三阶段一眼可分辨
- 顶部 HUD 信息不再分散
- 当前阶段的主视觉区域更明确

### v0.1.2-b 执行阶段游戏化

目标：把执行阶段做得像“冒险正在发生”。

范围：

- 重做执行阶段主面板为 Encounter Stage
- 展示当前房间、敌人强度、NPC、事件、掉落揭示
- 底部时间轴条可视化
- 增加 1x / 2x 节奏切换
- 增加昼夜切换与伤害反馈等轻量动画

验收标准：

- 玩家进入执行阶段时，主视线集中在中部舞台
- 不阅读长文，也能看懂“现在在打什么、危险多高、还剩几个卡槽”

### v0.1.2-c 规划阶段编排化

目标：把规划阶段做成真正的“战役编排界面”。

范围：

- 左侧背包三类 tab
- 中央时间轴编辑区强化轨道感
- 点击房间后显示镶嵌容量、已知怪物、收益预估、夜晚风险
- 底部显示 Day1~DayN 规划条
- 当前日、已锁定日、未来可编辑日视觉区分

验收标准：

- 玩家第一次进入规划阶段时，能直觉理解“左边选卡，中间排程，右边看结果”
- 昼夜分界和耐力占用无需解释即可理解

## 五、界面设计方向

### 1. 顶部 HUD

改造目标：

- 取消现在偏 dashboard 的均质五卡结构
- 改成更扁平、更聚合的资源栏

建议包含：

- HP 主血条
- 灵魂数
- 治疗瓶
- Boss 逼近度进度条
- 当前 Day / 阶段

要求：

- 扫一眼即可读完战斗资源
- 比标题和操作按钮更优先

### 2. 执行阶段中央舞台

改造目标：

- 将当前卡槽转为“遭遇场景”
- 展示勇者正在面对的对象，而非只展示卡牌说明

建议模块：

- 当前房间标题 + 场景描述
- 怪物区域：敌人类型 / 危险度 / 昼夜状态
- 友方区域：勇者 / NPC 支援 / 当前增益
- 揭示区域：掉落、随机事件、特殊触发

表现方式：

- 使用分层面板、图标、条形图、标签
- 保留文字，但以视觉模块为主

### 3. 当天推进时间轴

改造目标：

- 当前执行进度不再主要依赖右侧列表
- 时间轴成为执行阶段底部的核心反馈元素

建议状态：

- 已完成
- 当前执行
- 待执行
- 白天 / 黄昏 / 夜晚

建议表现：

- 卡槽块状推进条
- 当前块有动态高亮
- 夜晚区域以迷雾视觉覆盖

### 4. 规划阶段时间轴编辑区

改造目标：

- 从“卡片堆叠列表”转向“可理解的行程轨道”

建议表现：

- 显示每张卡占用时长
- 直观看到日夜切分线
- 显示剩余耐力
- 显示高品质卡对 Boss 逼近度的影响

### 5. 背包面板

改造目标：

- 强化分类浏览效率
- 减少一次性平铺造成的噪音

建议：

- 使用 `路程卡 / 镶嵌卡 / 角色卡` tab
- 每类卡保持统一信息密度
- 可放置、不可放置、冷却中状态用视觉而非长文区分

### 6. 右侧状态与预测面板

改造目标：

- 不只展示角色属性，也展示“当前编排的结果预测”

建议加入：

- 角色当前属性与装备
- 治疗瓶与背包容量
- 当前选中房间的已知收益 / 风险 / 可镶嵌容量
- 夜晚房间只显示模糊危险等级与未知信息

## 六、信息层级落地方案

### 1. 确定信息

内容：

- 房间基础效果
- NPC 功能
- 已装备效果
- 已知掉落池

展示方式：

- 明文 + 图标 + 标签

### 2. 模糊信息

内容：

- 难度指标
- 收益预估
- 夜晚风险

展示方式：

- 星级
- 进度条
- 风险标签

### 3. 隐藏信息

内容：

- 骰子结果
- 稀有掉落
- 随机事件具体结果

展示方式：

- 执行时动态揭示
- 规划阶段只显示“可能发现”“未知掉落”

### 4. 迷雾信息

内容：

- 夜晚房间怪物详情

展示方式：

- `?`
- 危险区间
- 未知提示文案

## 七、视觉风格要求

### 方向关键词

- 暗黑奇幻
- 余烬
- 石碑
- 铁锈
- 仪式感
- 编排命运

### 视觉原则

- 避免通用后台感
- 避免全界面均质卡片
- 主次区域必须明显
- 重点资源必须持续可见
- 不依赖大段文案表达状态

### 动效原则

- 只做少量有意义的动画
- 优先用于推进、切换、揭示、受击
- 不在高频重渲染容器上滥用入场动画

## 十、验收标准

`v0.1.2` 完成后，应满足以下标准：

- 玩家打开执行阶段时，第一感受是“正在冒险”，而不是“在看状态表”
- 玩家打开规划阶段时，第一感受是“在编排路线”，而不是“在管理卡片”
- 顶部 HUD 能在 1 秒内读完关键信息
- 白天 / 夜晚 / 当前执行 / 待执行 / 未知风险在视觉上明显区分
- 不看长文也能做出基础判断
- 整体界面气质更贴近 GDD，而不是 demo 面板

---

## 十一、不纳入 v0.1.2 的内容

以下内容本版本暂不纳入，以避免范围失控：

- 新增完整美术资源生产流程
- 稀有遭遇系统
- 祝福 / 诅咒系统
- 音效系统
- 全面重构为 Canvas / Phaser 表现层
- 新玩法规则扩展

这些内容建议放入后续版本。

---

## 十二、结论

`v0.1.2` 的重点不是“美化一下”，而是把当前已经能运行的玩法，用更符合 GDD 的方式表达出来。

如果 `v0.1.1` 解决的是“玩法闭环能不能跑”，那么 `v0.1.2` 要解决的是：

**它看起来、读起来、操作起来，像不像最初设计中的 DS18。**
