# RW01 - 少年宗门 Demo设计文档

## 项目信息

- **项目代号**: RW01
- **项目名**: 少年宗门
- **版本**: Demo验证版
- **核心验证点**: 订单系统 + 掌门等级循环
- **创建时间**: 2025-02-24

---

## 一、核心设计原则

### 1.1 资源缺口设计
> **灵石始终有缺口**，玩家需要在建筑升级、弟子培养、弟子招募之间做选择。

### 1.2 进度阶梯设计
> **完成X个订单后掌门升级**，X随境界递增，制造"越后期越需要努力"的进度感。

### 1.3 体验优先
> 掌门升级必须有**仪式感**，弟子必须有**身份感**。

## 二、系统架构

### 2.1 核心循环

```
┌─────────┐    完成订单    ┌─────────┐    达成条件    ┌─────────┐
│  订单   │ ────────────→ │ 掌门经验 │ ────────────→ │ 掌门升级 │
│  系统   │               │         │               │ 解锁内容 │
└─────────┘               └─────────┘               └────┬────┘
     ↑                                                    │
     │                                                    │
     │              ┌─────────────────────────────────────┘
     │              │
     │         解锁新建筑/新战场/新弟子上限
     │              │
     └──────────────┘
        弟子变强→采集更多材料→完成更多/更难订单
```

### 2.2 系统耦合图

| 系统 | 输入 | 输出 | 耦合系统 |
|-----|------|------|---------|
| **订单** | 背包材料 | 经验+灵石 | 掌门、背包 |
| **掌门** | 经验 | 解锁内容+升级特效 | 建筑、弟子、战场 |
| **弟子** | 灵石 | 战斗力 | 战场 |
| **战场** | 上阵弟子 | 材料→背包 | 背包、弟子 |
| **建筑** | 灵石 | 功能提升 | 战场、弟子 |
| **背包** | 战场掉落 | 材料存储 | 订单 |

```
│  战场 - 云雾山脉                                         │
├─────────────────────────────────────────────────────────┤
│  上阵槽位 (2/3)                                          │
│  ┌─────────┐  ┌─────────┐  ┌─────────┐                │
│  │ 💙李青云 │  │ 💙王浩然 │  │   [+]   │                │

│  │ 🐺野狼  │  │ 🐗野猪  │  │         │                │
│  │ HP: 45  │  │ HP: 12  │  │         │                │
│  └─────────┘  └─────────┘  └─────────┘                │
├─────────────────────────────────────────────────────────┤
│  累计掉落                                                │
│  🌿灵草×12  ⛏️矿石×8  🐾兽皮×3                          │
│                              [一键领取]                 │
└─────────────────────────────────────────────────────────┘
```

#### 战斗规则

1. **上阵**: 点击槽位选择弟子上阵
2. **战斗**: 自动进行，每1秒双方攻击1次
3. **胜利**: 怪物HP归0 → 掉落材料 → 自动刷新下一只
4. **战败**: 弟子HP归0 → 暂停战斗 → 开始恢复

#### 恢复机制

```
恢复时间 = (最大血量 - 当前血量) / 恢复速度

恢复速度 = 基础恢复 + 药坊加成 + 弟子等级加成
- 基础恢复: 2hp/s
- 药坊加成: 每级+3hp/s
- 弟子等级加成: 每级+1hp/s

示例:
- 练气期弟子(Lv.3)在Lv.2药坊中恢复:
- 恢复速度 = 2 + 6 + 3 = 11hp/s
- 满血恢复时间 ≈ 7-9秒
```

## 六、数值规划

### 6.1 经济循环

**目标**: 灵石始终有缺口，玩家需要做出选择

**收入**（每个订单周期约30秒）:
- 平均订单奖励: 150💎
- 每分钟收入: ~300💎

**支出选择**:
- 建筑升级: 300-2400💎
- 弟子升级: 100-3200💎
- 弟子招募: 500💎/次

**缺口设计**:
- 玩家无法同时升级所有内容
- 需要在"短期战力"和"长期发展"之间取舍

### 6.2 进度曲线

| 阶段 | 时间 | 目标 |
|-----|------|------|
| 0-5分钟 | 新手期 | 熟悉操作，完成第一个升级 |
| 5-15分钟 | 发展期 | 招募新弟子，升级建筑 |
| 15-30分钟 | 扩张期 | 解锁新系统，体验完整循环 |
| 30分钟+ | 长期 | 宗门飞升（Demo终点） |

### 6.3 关键数值表

#### 掌门升级

| 境界 | 需要订单数 | 累计订单 | 预估时间 |
|-----|-----------|---------|---------|
| 练气→筑基 | 3 | 3 | 2分钟 |
| 筑基→金丹 | 5 | 8 | 4分钟 |
| 金丹→元婴 | 8 | 16 | 7分钟 |
| 元婴→化神 | 12 | 28 | 12分钟 |
| 化神→飞升 | 20 | 48 | 20分钟 |

**Demo总时长**: 约45分钟体验完整循环

#### 弟子等级上限

| 掌门境界 | 弟子等级上限 |
|---------|------------|
| 练气期 | Lv.3 |
| 筑基期 | Lv.5 |
| 金丹期 | Lv.8 |
| 元婴期 | Lv.12 |
| 化神期 | Lv.15 |

## 九、备注

### 9.1 版本规划

**RW01 Demo目标**:
- 验证核心循环是否流畅
- 验证掌门升级是否有成就感
- 验证资源缺口是否驱动决策

**后续版本预留**:
- 武器/防具打造系统
- 多战场选择
- 弟子技能
- 宗门飞升完整机制

### 9.2 测试要点

1. **首次升级体验**: 是否在3分钟内完成第一次掌门升级？
2. **资源决策**: 玩家是否会纠结"先升建筑还是先升弟子"？
3. **订单节奏**: 订单刷新速度是否合适？不会积压也不会空等？
4. **战斗参与感**: 自动战斗下玩家是否愿意盯着看？
